Please upgrade to the current release (6.3.17) and read section G3 of
the FAQ (http://fetchmail.berlios.de/fetchmail-FAQ.html#G3).  Note
that something must have changed to cause the failure, and if the
change wasn't with Fetchmail then it almost certainly relates to
either the Exchange server, or the account used to authenticate with.

> fetchmail: POP3< +OK Begin TLS negotiation.
> fetchmail: Server certificate verification error: unable to get local issuer certificate
> fetchmail: Server certificate verification error: certificate not trusted

You presumably didn't test TLS by hand ;)  I'd recommend you resolve
this error (http://fetchmail.berlios.de/fetchmail-FAQ.html#K5) since
it may be related to your problem.
