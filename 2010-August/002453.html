Am 19.08.2010, 00:59 Uhr, schrieb Globe Trotter:

> Hi,
>
> I have been trying to set up fetchmail to fetch e-mail from a MS  
> Exchange 2007 IMAP4 server and then read it locally. So here is my  
> .fetchmailrc file.
>
> set daemon 600
> poll  exchange.uia.edu with proto AUTO ssl
> mda 'procmail -f-'
> keep
>
> I invoke fetchmail using the command:
>
> %fetchmail
>
> This "works" (after supplying password) except that if I try this from  
> another machine, I get that I have no messages that are not seen.  
> Specifically, I get that all the kept messages have been seen. How do I  
> fix this?

If you're actually using IMAP (check the logs, you're using proto AUTO),  
you cannot do that, because fetchmail currently relies on the server-side  
\Seen flags to track which messages were seen or not.  A change is planned  
for a future version, but the code does not yet exist.

If fetchmail were to pick POP3 because IMAP was blocked somehow, you could  
work around that by using the uidl option.
