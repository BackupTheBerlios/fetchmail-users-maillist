Encrypted with a password, but the root user can intercept them. Either by  
installing keyloggers to the encryption password, or by sniffing the NSS  
libraries before passwords are encrypted, or from process memory.

> Secondly, I don't see why having a key (to decrypted passwords) isn't  
> more secure.  They would need physical access to my USB drive.
> Plug it in, download mail, unplug it.  The opportunity of attack is much

And the USB drive has which filesystem? Microsoft FAT perhaps? :-)

Anyways, Ben Kibbey has contributed a fetchmail branch that supports PWMD  
(password manager daemon). The description is at  
http://benkibbey.wordpress.com/pwmd/ and the patched fetchmail source tree  
is at http://gitorious.org/~bjk/fetchmail/fetchmail-libpwmd/trees/master.  
You can download a tarball, but it's not preconfigured the same way as the  
release tarballs (because they are snapshots from the repository), so need  
to have some more tools installed than usual, namely, autoconf, automake,  
flex, gettext-devel (depending on your distribution), bison, m4, and you  
need to run ./autogen.sh before you can run ./configure.

If that helps, let me know your experiences.  I haven't felt the need or  
had the time to try it, but I still plan to merge it for the next feature  
release if it works well in practice.
