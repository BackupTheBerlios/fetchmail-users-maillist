On Sun, Aug 1, 2010 at 20:47, Depo Catcher <depocatcher at gmail.com> wrote:
>
> I have fetchmail working for a dozen or so sites (it's really nice btw;
> works great).  The only problem is now my .fetchmailrc file has a dozen or
> so passwords in plain text.

If this isn't in the FAQ it'll be covered many, many, many times in
the list archive.

> What's the best way to tighten this up a bit?
>
> At work we have a in house application that stores passwords in an encrypted
> database.  The way you put passwords in scripts is you use a key file to
> "PasswordVault -username test -getpassword -key /mykey.txt" which returns
> plain text password.
> Not much more secure; but would help from a `cat ~/.fetchmailrc` "attack".
>
> Anything like that in fetchmail or third party?

No, because it buys you no meaningful security.  Anybody who could
have otherwise got access to your password by reading the file can run
the tool and get the password.  You've bought yourself nothing.

To stop the "attack" you mention, ensure only the owner can read the
file (fetchmail will refuse to run if this isn't the case).  If you
don't trust the root user then you have bigger issues.
