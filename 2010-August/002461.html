Alan,

can I ask you to do me a favour and test a revision that I hope fixes
this problem?

If yes,

1. please download
<http://homepages.upb.de/mandree/fetchmail-6.3.17+gss1.tar.bz2>

2. unpack the tarball

3. ./configure -C --with-ssl --with-gssapi && make -j6

4. If you have ever run "kinit" before, please run "kdestroy" before
trying fetchmail.

5. Then run the newly built ./fetchmail version, and please report -
with a "fetchmail -vv" log - if Exchange still fails without
credentials, or if it proceeds to trying the password right away
successfully.

If this succeeds, the changes will become part of fetchmail 6.3.18, and
I will also try to have the Debian maintainers backport the changes - or
at least include them in spite of the freeze.
