> I take this to mean that setting smtpaddress as above isn't enough to
> not make fetchmail send an helo different than localhost. Is this correct,
> and if yes, isn't there any other way to make fetchmail helo something
> else?

Right, having taken a moment to dive into the code, the value of the
HELO/ELHO string that is presented is the value of smtphost.  This
defaults to localhost - change this to the fully qualified name of the
host and ensure that it is configured to NOT reject connections that
provide it's own name as the HELO value.

Note that rejecting unqualified HELO values while being fully RFC
compliant will trip up a large number of MUAs (particularly those from
Microsoft).  If you don't expect any mail clients to ever connect
directly you'll probably be ok.
