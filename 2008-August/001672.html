This came up recently, but in short:

1) Install the latest fetchmail (6.3.8).  You *can* copy the binaries
from a server with an identical base OS and update level, but it's
probably best to do it fresh.

2) Copy across the fetchmailrc file(s) and any fetchid file(s) for POP3.

3) Run a "fetchmail --nosyslog --nodetach -vvv -c" to see if it
correctly connects to the remote server(s) and identifies unread mail
as you expect (this won't pull down any email).

4) Check that your local mail delivery works as expected ;)

5) Shut fetchmail down on the old server, start it up on the new one.
