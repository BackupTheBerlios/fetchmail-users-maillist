Hi,

On 03.09.2012 22:30, Johannes Bauer wrote:

> I'm using fetchmail together with Dovecot (and Dovecot's "deliver") on
> multiple (approx. 30) accounts with no problems at all. A new account
> recently was added with a new server, and it's not fetching mails from
> that one. [...]

> poll $server
> 	protocol pop3
> 	user "$user"
> 	password "$password"
> 	ssl
> 	mda "/usr/lib/dovecot/deliver"
> 	no keep

I guess this is the problem:

> fetchmail: POP3> STAT
> fetchmail: POP3< +OK 2 12488
> fetchmail: POP3> LAST
> fetchmail: POP3< +OK 2
> 2 messages (2 seen) for $user at $server (12488 octets).
> skipping message $user@$server:1 not flushed
> skipping message $user@$server:2 not flushed

The server tells the last message that has been read already is 2 (out
of 2), so fetchmail skips them. AFAIK the LAST command has been
abandoned long ago.

Try to configure "fetchall", you will be deleting them from the server
anyway ("no keep").

Robert
