> > msmtp: could not send mail (account default from /home/nbecker/.msmtprc)
> > fetchmail: MDA returned nonzero status 65

>
> Ah, I just noticed you're using the MDA option.  There is (that I know
> of) no way for fetchmail to get anything other than the return code
> from the MDA.  The return code will be little more than success or
> failure.
>
> You need to either have your local system accept email from <> (which
> by the RFCs you're not supposed to reject mail from - see
> http://www.rfc-ignorant.org/policy-dsn.php) or use an (E)SMTP
> listener.

I think I solved it.  I didn't know that I didn't need a local mda to relay on 
to my server.  Now it looks like:

imap server -> fetchmail -> my remote server

I use:

poll upstream_imap_server proto imap user upstream_user at domain password xxx 
smtphost 10.4.0.1 smtpname user at remote_host_name

This sends directly to my remote server, rewriting the rcpt to be 
user at remote_host_name.

Perhaps add this to a FAQ?
