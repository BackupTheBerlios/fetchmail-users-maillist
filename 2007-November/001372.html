On 11/4/07, James Nunnerley <james at webit.co.uk> wrote:
> I've started using fetchmail for the first time.  The intention is to use it
> to retrieve emails so that they then get checked by exim (which includes
> spam and anti-virus checking) and it then forwards onto an Exchange server.
> I have now got it working, but stupidly first time I ran it, selected the
> fetchall option.  That first time it wasn't setup correctly.  Can anyone
> advise me where the emails will have gone based on the following config
> file:
>
> poll 'mail.server.com'
>      user 'username'
>      pass 'password'
>      is lucy
>
> I presumed it would go to user lucy - but on checking /var/spool/mail/lucy,
> there are none there... can anyone help?

As Peter said, fetchmail will have handed the emails over to your
local SMTP server - check the mail logs to see what it did with them.

Note that the default location for local emails has more to do with
your choice of POP/IMAP server (and associated LDA) than your SMTP
server.
