> fetchmail: MDA returned nonzero status 65

Ah, I just noticed you're using the MDA option.  There is (that I know
of) no way for fetchmail to get anything other than the return code
from the MDA.  The return code will be little more than success or
failure.

You need to either have your local system accept email from <> (which
by the RFCs you're not supposed to reject mail from - see
http://www.rfc-ignorant.org/policy-dsn.php) or use an (E)SMTP
listener.
