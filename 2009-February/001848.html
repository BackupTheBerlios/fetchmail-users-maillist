Ken Gillett schrieb:
> So if I start it with:
> 	fetchmail -N -d 3600
> 
> and then sometime later just run:
> 	fetchmail -N -d 60
> 
> Would that simply change the polling frequency of that first instance  
> of fetchmail?

No, sorry - it's not as convenient as that. fetchmail cannot pass options
except "quit" or "re-poll" (which are in fact just signals) to running
instances.

You could try changing the configuration file to "set daemon 60" and hope
that the daemon re-execs itself successfully. It's supposed to do that, so
let me know (along with instructions how to reproduce) if it fails.

BTW, note that fetchmail 6.3.9 has been released 2.5 months ago and fixes
some critical and security bugs that 6.3.8 showed. I meant to wrote that in
an earlier message, but forgot. Sorry.
