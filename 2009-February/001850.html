I was thinking that maybe I'd run fetchmail separately for each user  
on the server, each with their own rc file so quite independent, but  
both with a default poll interval of say 3600 (1 hr). Then on the  
client machine, arrange launchd to run a more frequent rsh command to  
awaken the daemon and get more frequent polling, until the user logs  
out and stops the rsh command and it reverts back to the default  
interval.

Not sure how to cope with multiple simultaneous logins by the same  
user on different machines yet, still thinking about it. Also, am I  
right in thinking that when a user runs fetchmail to awaken the  
daemon, it will only awaken that one user's instance of fetchmail and  
any other instances will be unaffected?
