>>> b) Ask fetchmail to retry without having it fetch the same mail
>> repeatedly?
>>> For instance, let us say I said "fetchmail -ak" and it failed at 13894th
>>> email. I'd like to restart it and have it start fetching from the 13895th
>>> email. Can this be done? How?
>> ...this isn't too helpful with IMAP4 since fetchmail cannot track which
>> mails it has seen or not in this case, POP3 UIDL (then without -a) would
>> likely work better.
> 
> 
> Using POP3 isn't an workable alternative - since our exchange server's POP3
> support is notoriously flaky. I have to rely on IMAP4.

Maybe, but if you start with an empty .fetchids, "fetchmail --uidl --proto
pop3 --keep" will be able to track by itself which mail it has seen, and
also fetch all. If it gets interrupted, restart and it will pick up where
it left off, with the overhead of retransmitting the UID list (one line per
mail, typically on the order of magnitude of 15 - 50 bytes) and fetching
the mail that was jammed.
