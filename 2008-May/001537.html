Hello,


I'm new to fetchmail and also new to this list and have a question concerning fetchmail and the 
"cooperation" with postfix.

Used system is a Mac mini running Mac OS X.4.11 with fetchmail and postfix. I'm trying to get mails 
from my external account and would like to see this mail in my normal inbox.

I've set up postfix to and can see there mails which are generated by cron. So this seems to work. 
But mails fetched by fetchmail don't appear in the inbox. I use under my normal user account a 
.fetchmailrc with the following contents:

poll pop3.web.de proto pop3 user Michael.Homscheidt at web.de with password <remotepassword> is homsch 
here

In the log file I see that fetchmail processes a number of emails; this number is the same number of 
unread emails under the webinterface for Michael.Homscheidt at web.de.

In contrast I woudl expect that the file /var/mail/homsch changes its contents after fetchmail has 
done its job. But this remains untouched.

What's wrong with my configuration?
