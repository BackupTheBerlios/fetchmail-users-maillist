On Tue, Mar 18, 2008 at 11:06 AM,  <isabel at fbmsoftware.com> wrote:
> Hi everyone!
>
>  I hope you could help me guys.
>
>  This is my setup:
>
>  *       I'm running fetchmail in a Debian machine.
>  *       The Debian machine has 2 different IP addresses (IP1 and IP2).
>  *       Fetchmail is using IP1 to connect to the server (mail.something.com)
>  where we get our emails. I use the "netstat -tupan | grep fetchmail" command
>  to confirm this.
>
>  My problem now is, the server we are accessing (mail.something.com) only
>  allow IP2 to pass through their server but fetchmail is using IP1. Is there
>  anyway to force fetchmail to bind or use IP2 instead and not IP1? I want to
>  try first if this is possible. If I have no choice, I would ask the server
>  (mail.something.com) we are accessing to allow our other IP.

This is really just a network routing issue - simply add a static route:

 route add -host mail.something.com ethX

Replace "ethX" with the name of the interface that IP2 is bound to.
Add this to the appropriate config file (sorry, I don't know enough
about Debian to say for certain how to do that).
