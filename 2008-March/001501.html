On Mon, Mar 31, 2008 at 4:40 PM, Haines Brown <brownh at hartford-hwp.com> wrote:
> Periodically I get an error from my ISP's mail server. I suspect it is
>  really just a warning, for it does not prevent the mail from being
>  retrieved. I'm running fetchmail on debian etch with exim.
>
>  $ fetchmail -cvv
>  ...
>  fetchmail: POP3< +OK Begin SSL/TLS negotiation now.
>  fetchmail: Issuer Organization: Courier Mail Server
>  fetchmail: Issuer CommonName: localhost
>  fetchmail: Server CommonName: localhost
>  fetchmail: Server CommonName mismatch: localhost != pop.hartford-hwp.com
>  fetchmail: pop.hartford-hwp.com key fingerprint: 27:33:38:C0:92:FF:CE:37:E
>  2:BC:70:7C:25:24:E5:03
>  fetchmail: Server certificate verification error: self signed certificate
>  fetchmail: POP3> CAPA
<---SNIP--->
>  I need an explanation of the error message lest it is a result from a
>  misconfiguration.

The error is just telling you that the certificate wasn't signed by a
CA, but by the creator of the certificate.  It's warning you that you
need to pay more attention as, ultimately, the certificate isn't
signed by an organisation that's generally trusted (such as Verisign).

The simple solution is to make use of the fingerprint:

poll my.isp.com
  user remote with password insecure
  ssl fingerprint "27:33:38:C0:92:FF:CE:37:E2:BC:70:7C:25:24:E5:03"
  is myself here

The first google hit on the error message would have got you this if
you read the thread ;)
