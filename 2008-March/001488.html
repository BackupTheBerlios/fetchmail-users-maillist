Stuart J. Browne schrieb:
> Hi,
> 
> I'm using the 'antispam' option to drop messages with a few different errors
> thrown back from the local SMTP, but unfortunately there are two responses
> which are almost identical.
> 
> The '451' response seems to be used as a general soft-failure (by sendmail),
> giving secondary error numbers for further detail, i.e.:
> 
> fetchmail: SMTP error: 451 4.1.8 Domain of sender address
> _chiu at acenteiletisim.com does not resolve
> fetchmail: SMTP error: 451 4.3.2 Please try again later
> 
> Is it possible for fetchmail to take this second number into consideration?
> I'd like the second one to be ignored (as it's due to a temporary milter
> failure, message shouldn't be flushed), and the first to be acted upon (non
> existent domain, message should be flushed).

While fetchmail might be modified to do that (which requires some more 
patching though), Sendmail might also issue 451 4.1.8 for a temporary 
failure of legit and solicited mail, so I don't think it's a good idea to 
do that.
