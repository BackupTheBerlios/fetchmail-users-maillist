> 
> poll mail.foo.com protocol pop3 uidl:
>   user "nelson" password "abc123" is "nelsonts2" here options keep;
>   user "nelson" password "abc123" is "nelsonts" here options keep;
> 
> Then I run the command fetchmail -v -U -k -S localhost/587.

Fetchmail assumes that the user at server name is unique, but neither
validates nor enforces that.  Your configuration violates that
assumption and malfunctions in the observed way.

I suppose that when reading back messages, all UIDs read from both user
lines are actually stored for the first nelson at mail.example.com poll.

I won't add a relevant check to fetchmail 6.X though, and probably not
to 7.X either. I might do something about this in 8.X, but that's a long
way from now.
