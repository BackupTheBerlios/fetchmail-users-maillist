I have setup fetchmail to fetch from a multidrop pop account and deliver to a local MTA.
   
  Some messages (mostly spam or corrupted) cause the MTA to close the socket and not accept the message. In this case fetchmail will retry to deliver the same message again and mails will get accumulated until I delete the problematic messge from the queue.
   
  Is there a way to avoid this issue and let fetchmail simply delete the message or ignore it or deliver it to a local spool.
   
  My .fetchmailrc file as follows:
   
  set daemon 120
set logfile "/var/log/fetchmail"
set postmaster postmaster at domain.com
set no bouncemail
poll mailserver.domain.com
        proto pop3
        localdomains domain.com
        envelope 1 Delivered-To:
        qvirtual '17-'
        no dns
        user "popuser"
        pass "password"
        to * here
        fetchall
        no keep
   
  Thanks
