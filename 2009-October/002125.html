Please run

env LC_ALL=C fetchmail -vvv --nosyslog --nodetach     # (add other options  
if needed)

to produce English-language error messages. This is documented in

http://www.fetchmail.info/fetchmail-FAQ.html#G3

which please follow for all further problem reports.


Please also check your mailer configuration. You have pasted Unicode  
(UTF-8) characters but your mailer declared ISO-8859-1 character set,  
destroying accented characters.


With the little Spanish I know I understand the following:

1. fetchmail complains about being run as root - please check if your MDA  
is designed in a way that it's safe and secure to be set-uid root, and if  
so, run fetchmail as unprivileged user (best to create a new group and  
user) and make your MDA set-uid root.
       I'm not using Dovecot's deliver and never bothered to check. I know  
maildrop works this way. Procmail is also specified to work that way, but  
I don't trust it, and procmail is unmaintained.

2. the MDA exits with code 75, i. e. temporary failure. That is not a  
fetchmail problem, but a problem of your MDA (deliver I presume after  
reading your other mail) - please check what your MDA logs and if it's  
Dovecot deliver, please ask on the Dovecot mailing list. fetchmail  
consequently leaves messages on the server.
