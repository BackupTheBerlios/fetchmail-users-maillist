Ok I figured it out.  Based on this email thread,
https://lists.berlios.de/pipermail/fetchmail-users/2008-September/001708.html 
I ran the strace command and saw that fetchmail was looking for a
different CA than the one I downloaded from using the openssl command. 
Specifically it was looking for the file "Equifax_Secure_CA.pem" which
was housed in my global certificate store (/usr/lib/ssl/certs).  So I
just copied that file to the local certificate store I had specified in
my .fetchmailrc and ran c_rehash to regenerate the store.  Then things
started working.
