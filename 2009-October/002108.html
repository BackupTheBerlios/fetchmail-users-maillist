> Received: from ?????? ([211.225.33.79])
>         by mx.google.com with ESMTP id

> permitted sender) smtp.mail=abfsupajsih@\ufffd\ufffd\ufffd\ufffd

...that seems to confirm it. The sending domain is just crap, \ufffd is  
the Unicode replacement character and hints to earlier encoding errors.

> I am not sure why fetchmail is not simply delivering the message.

It tries to, but your SMTP server (MTA) refuses it - and rightfully so.  
Nothing fetchmail can do about that, and Google shouldn't be accepting or  
sending such messages.

Permanently kill the message with the invalid domain through the web  
interface and move on.

> Is the some configuration setting that I should be using?

Perhaps you may want to "set nosoftbounce" so that such permanently  
undeliverable messages get removed.

First and foremost, as written above, the upstream server where you're  
fetching from shouldn't accept such nonstandard garbage.
