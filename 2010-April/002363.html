> 
> Fetchmail can use syslog, why not use syslog?
>

Well, as long as you are serious. ;)

Syslog is not a dedicated log. It certainly works; however, it also
contains a lot of information that is not Fetchmail related. Again,
while I could create scripts to filter out non-fetchmail data, I
believe that a simple, dedicated log file would be more convenient and
easier to inspect.

All of my other daemons like clamav, denyhosts, ipfirewall, mysql, etc
create dedicated logs. I just think that a dedicated format is a better
idea.

All of the daemons I mentioned, and several I haven't have the ability
to receive signals that allow their logs to be rotated without
incidence. I would think that it is doable in Fetchmail as well. Then
again, I have been known to be wrong on rare occasions! :-D
