On Sun, 25 Apr 2010, Rob MacGregor wrote:

> On Sun, Apr 25, 2010 at 01:23, Robert Holtzman <holtzm at cox.net> wrote:
>> I added "set syslog" to my .fetchmailrc files. The first thing that I noted
>> was that the download messages that used to be written to the terminal were
>> instead written to the log files. The second thing was that it is a bad idea
>> to write to /var/log/syslog because the the output gets mixed up with all
>> the other log entries. In addiion, the log messages are written to
>> /var/log/mail.log. That makes sense but I cannot find a way to 1) write to
>> /var/log/mail.log *only*
>
> That's caused by your syslog configuration.  Exactly what you do
> depends on your version of syslog and what operating system you're
> running.  In general the configuration file is /etc/syslog.conf and
> you'll either want to remove "mail.*" or add "!mail.*" to the line
> containing /var/log/syslog.  As I suggested previously, try switching
> to syslog-ng which you'll probably find better suits you since you'll
> be able to keep all of fetchmail's logging in a single file.
>
>> and 2) have the download messages appear on the
>> terminal also.
>
> The only way to do that is to run "tail -f /var/log/mail.log".

That did it. Thanks.
