Hi all

 

I'm using fetchmail 6.3.6-1etch1 + postfix  2.3.8-2 on debian etch. In my
postfix main.cf file, smtpd_tls_security_level = encrypt is set like below. 

 

disable_vrfy_command = yes

smtpd_sasl_type = dovecot

smtpd_sasl_path = private/auth

smtpd_sasl_auth_enable = yes

smtpd_recipient_restrictions = permit_auth_destination, check_sender_access
mysql:/etc/postfix/mysql-virtual-dismail-maps.cf,permit_sasl_authenticated,
reject_unauth_destination

smtpd_use_tls = yes

smtpd_tls_auth_only = yes

smtpd_tls_security_level = encrypt

smtpd_tls_loglevel = 1

smtpd_tls_received_header = yes

smtpd_tls_session_cache_timeout = 3600s

smtpd_tls_cert_file=/etc/ssl/certs/dovecot.pem

smtpd_tls_key_file=/etc/ssl/private/dovecot.pem

smtpd_tls_session_cache_database = btree:${queue_directory}/smtpd_scache

tls_random_source = dev:/dev/urandom

smtpd_helo_required = yes

broken_sasl_auth_clients = yes

 

 

my fetchmailrc is like that;

 

poll mail.myisp.com proto pop3 user username at mydomain.com pass "mypass" is
username at mydomain.com here

 

 

my problem is; when I start fetchmail , it gives me a error regarding TLS
like as below.

 

fetchmail: reading message username at mydomain.com @ mail.myisp.com:1 of 1
(3077 octets) (log message incomplete)fetchmail: SMTP error: 530 5.7.0 Must
issue a STARTTLS command first

 

But if I change smtpd_tls_security_level parameter in main.cf from encrypt
to may. It works fine. As I understood by this error, fetchmail can not use
TLS while it delivers mail to the local MTA.

 

So how can I solve this problem without changing smtpd_tls_security_level
parameter? 

 

Best regards
