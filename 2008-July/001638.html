<---SNIP--->
> my problem is; when I start fetchmail , it gives me a error regarding TLS
> like as below.
>
> fetchmail: reading message username at mydomain.com @ mail.myisp.com:1 of 1
> (3077 octets) (log message incomplete)fetchmail: SMTP error: 530 5.7.0 Must
> issue a STARTTLS command first
>
> But if I change smtpd_tls_security_level parameter in main.cf from encrypt
> to may. It works fine. As I understood by this error, fetchmail can not use
> TLS while it delivers mail to the local MTA.
>
> So how can I solve this problem without changing smtpd_tls_security_level
> parameter?

Can you not configure postfix to make it optional only for the
loopback itnerface?  There's no actual network traffic going on, so no
security issue (and, unless every other link in your email chain
supports SSL/TLS then there's little security issue in not supporting
SSL/TLS for that final hop anyway).
