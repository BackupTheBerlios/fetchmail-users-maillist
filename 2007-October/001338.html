Hello list members

My current task is to migrate the old commercial product Intermail
PostOffice (SMTP/POP3 Server for Windows) to a postfix/dovecot
environment. To copy the mails from one server to the other I decided to
use fetchmail. Now comes the problem:

The existing users have very much eMails stored on this server and I
don't want them to redownload all of them (cause the POP3-UIDL has changed).

Dovecot has the nice option "pop3_reuse_xuidl"
(http://wiki.dovecot.org/MainConfig?highlight=%28pop3_reuse_xuidl%29) to
look in all new eMails for the header "X-UIDL" and if found, use its
value as the UID. This works.

Now my question:
How can fetchmail put the header "X-UIDL" in all eMails it gets from a
pop3 account? Or is there an alternative project which is able to do that?

Many thanks in advance for your help.
