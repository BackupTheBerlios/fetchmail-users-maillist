On 10/18/07, Joe Acquisto <joea at j4computers.com> wrote:
> I am currently using fetchmail to pop several mailboxes and deliver them to a Groupwise system.  Fairly simple, one user per POP mailbox.
>
> I would like to setup another system, where fetchmail would pop from two POP accounts (diffferent domains).
>
> Each of those POP mailboxes would be configured to hold many (several) different user names, within a given domain).  ex - joe at someplace.com, jim at someplace.com, nancy at someplace.com  The mailbox (user name) might be, say, group at someplace1.com .  The other would be, say, group at someplace2.com
>
> Further,  I would want to deliver each to a separate SMTP server (GWIA's actually).  One would be on the fetchmail host, the other on a separate box.
>
> Can fetchmail do this successfully?

Yes:

poll pop.ispA.net envelope Delivered-to user fred pass 1234 to * here smtphost X
poll pop.ispB.net envelope Delivered-to user mark pass 4567 to * here smtphost Y

See the man page for details on the correct configuration for multi-drop.
