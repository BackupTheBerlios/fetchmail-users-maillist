On 10/22/07, Joe Acquisto <joea at j4computers.com> wrote:
> This example for multidrop "poll pop.ispA.net envelope Delivered-to user fred pass 1234 to * here smtphost X", got me to wondering if the "*" character could be used in other situations.
>
> Such as allowing a lazy admin to reduce keystrokes.  In other words, if there are numerous lines, to poll numerous users or hosts, can in be used in place of an actual user, as a wildcard?

The second you use '*' for delivery you're using the multidrop
configuration.  While it's certainly possible to use this against a
singledrop mailbox, all the standard warnings for multidrop apply.
