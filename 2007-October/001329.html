The error relates to problems with the communications with the remote
(POP) server.  Reducing the maximum message size simply decreases the
chance of this problem striking while you're downloading a message.

> Perhaps you should consider (unrelated to this error) to add a configuration
> value to fetchmail to tell it to continue fetching other messages (e.g. skipping
> the problematic mail) rather than getting stuck on 1 mail and stopping the
> entire workflow.

You could use the "limit" keyword to skip messages over the specified
number of bytes.
