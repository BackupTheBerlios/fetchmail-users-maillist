I'd like to chime in with the "mail is not for file transfer" choir though.

So compile fetchmail, gcc users should use CFLAGS="-O -ggdb3", run it under
a debugger, put breakpoints on every line that returns PS_SOCKET (sorry, I
don't have a list, grep -n PS_SOCKET *.c on the fetchmail source should
help), and if a breakpoint is hit, obtain a stack backtrace, and at the
same time, log the network connections with tcpdump or wireshark/tshark so
we can see how fetchmail's behaviour relates to what goes over the wire -
that might be a constructive approach, but I'm not making promises here, it
might very well be the problem is still beyond any application's powers.
