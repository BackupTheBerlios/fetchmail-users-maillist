Hi Wade,

I'm sorry, but I'm not sure I understand the problem correctly - apparently 
there are multiple problems - take care not to confuse them.

Does using --uidl on the command line (or as "uidl" server option in the 
run control file, .fetchmailrc) help?

If there are 873 "seen" messages at gmail.com, how come 100 are missing? 
Have they been downloaded by another client? Apparently the log messages 
shown are incomplete.

About the login issue, I cannot help, that's a question for the Google support.

About the "Bad sender address syntax", the offending sender isn't shown in 
the log, but your SMTP server (Sendmail, Exim, Postfix are typical 
candidates) refused to accept the message.

Can you run

fetchmail -Nd0 --nosyslog -V

and

fetchmail -Nd0 -vv --nosyslog

and show the output you're getting? Check the first set for passwords and 
remove if found.

Feel free to edit repeating lines of same content.
