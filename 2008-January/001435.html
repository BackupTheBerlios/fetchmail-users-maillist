> I am running Debian sarge. I have installed fetchmail and some openssl
> and certifcates packages.
>
> I am attempting to use fetchmail to fetch mail from mail server
> pop.mail.yahoo.co.uk using SSL. Fetchmail does not like the mail
> server's SSL
> certificate. The output from any of the commands
>      fetchmail -c -v pop.mail.yahoo.co.uk
>      fetchmail -c -v --sslcertpath /etc/ssl pop.mail.yahoo.co.uk
>      fetchmail -c -v --sslcertpath /etc/ssl/certs pop.mail.yahoo.co.uk
> includes
>
> fetchmail: 6.2.5 querying pop.mail.yahoo.co.uk (protocol POP3) at Tue 22
> Jan 2008 18:39:51 GMT: poll started

Please note that you're using an unsupported version (as stated on the
fetchmail home page).

> I do have some certificates from Equifax because when I run (as root)
> c_rehash
> its output includes
> Equifax_Secure_CA.pem => 594f1775.0
> Equifax_Secure_eBusiness_CA_1.pem => e7b8d656.0
> Equifax_Secure_eBusiness_CA_2.pem => 8f7b96c4.0
> Equifax_Secure_Global_eBusiness_CA.pem => 74c26bd0.0

Have you tried adding/updating the ca-certificates package (apt-get
install ca-certificates)?

> I have tried the command
>      openssl s_client -connect pop.mail.yahoo.co.uk:995 test.fil
> This printed

Try instead the following 2 commands:

openssl s_client -connect pop.mail.yahoo.co.uk:995 < /dev/null | sed
-ne '/-BEGIN CERTIFICATE-/,/-END CERTIFICATE-/p' >
/etc/ssl/certs/pop.mail.yahoo.co.uk.pem
c_rehash

Though you're better installing/updating ca-certificates (if that works).
