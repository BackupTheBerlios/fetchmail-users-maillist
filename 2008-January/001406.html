drbob schrieb:

> Is there any workaround for this that would allow me to leave messages
> on the server without having them bounce after 30 days? Some way to
> automatically delete messages older than 29 days would work.

Recompile with the attached patch (WARNING: untested and undocumented) and 
set FETCHMAIL_POP3_FORCE_RETR=1 in the environment when running fetchmail, 
that should force fetchmail to use RETR instead of TOP - but it applies to 
all POP3 servers you're polling.
