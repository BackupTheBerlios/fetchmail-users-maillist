Hi Andrea,

the "LAST / -ERR Invalid command" is harmless, fetchmail uses UIDL to 
compensate (it's a better choice anyways, but for compatibility with 
fetchmail 6.2.X, we try LAST before trying UIDL).

What I find strange is: fetchmail considers the messages "unseen",
but does not apparently try to fetch them. Does "fetchmail -vvv" provide 
any further insight? Please try and show the log.
