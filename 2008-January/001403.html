Hello,

I use fetchmail to collect mail from my isp pop3 server. I would like to
leave mail on the server (keep) so that it can also be occasionally
checked from my pda or phone. However my ISP mail sever automatically
bounces kept messages older than 30 days if they have not been retrieved
with the RETR command. Fetchmail defaults to using TOP to collect mail.

 From the manual and previous discussion on this list
<http://lists.berlios.de/pipermail/fetchmail-users/2006-March/000293.html>
it seems that there is no way to force retrieval with RETR other than
the "fetchall" option, even if the server supports uidl.

The fetchall option is unsatisfactory as I don't want duplicate emails
every time I collect mail.

Is there any workaround for this that would allow me to leave messages
on the server without having them bounce after 30 days? Some way to
automatically delete messages older than 29 days would work.

Thanks in advance for any help.
