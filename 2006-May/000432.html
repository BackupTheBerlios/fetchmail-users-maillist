Filippo Santovito <filippo.santovito at email.it> writes:

> Hi all,
> I have a remote pop account (remote at popserver) that I need to duplicate into 
> two local mail spool (account1 at local, account2 at local);
> I tried with:
>
> #############################################
> poll popserver with proto POP3
> user 'remote at popserver' there with password 'xxxx' is 'account1' here options 
> fetchall ssl
>
> poll popserver with proto POP3
> user 'remote at popserver' there with password 'xxxx' is 'account2' here options 
> fetchall ssl nokeep
> #############################################
>
> but only account1 spool is created and filled with emails.
> How can I duplicate the remote account?

If the local machine has a sendmail-compatible sendmail command
(Postfix, Exim also count here), try:

poll popserver with proto POP3
user 'remote at popserver' there with password 'xxxx' 
options fetchall ssl
mda '/usr/sbin/sendmail -i -f %F -- account1 account2'

Just be sure none of account1 and account2 forward back onto
remote at popserver. Note that mda has some other implications -- see the
man page.
