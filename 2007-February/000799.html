Greetings,

The security fixes in 6.3.6 also had side effects, and broke these two
features:
- - KPOP authentication
- - repolling without TLS if opportunistic upgrade to TLS failed for POP3

I have uploaded the fetchmail 6.3.7 release candidate to the usual download location:
<http://home.pages.de/~mandree/fetchmail/> - this should become the
final 6.3.7 release quite soon - unless further regressions or critical
bugs are reported.

++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++++
fetchmail 6.3.7 changes since 6.3.6:

# FIXES FOR REGRESSIONS IN 6.3.6
* Fix KPOP. Patch by Miloslav Trmac.
* Fix repoll when server disconnects after opportunistic TLS failed for POP3.
  Berlios Bug #10133, reported by Andrej Kacian.

# TRANSLATION UPDATES
* Japanese (Takeshi Hamasaki), Polish (Jakub Bogusz)
