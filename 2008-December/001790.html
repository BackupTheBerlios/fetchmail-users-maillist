Burkhard Masseida schrieb:
> Dear fetchmail list,
> 
> i have a problem with my fetchmail fetching mail from a specific server
> with a unique UIDL scheme, using the keep-option.
> 
> I'm using the fetchmail binary provided by the Debian etch
> distribution (version 6.3.6-1etch1), but tried a self-compiled fetchmail
> 6.3.9 as well.
> 
> The problem is, that fetchmail seems to be unable to remember those
> special UIDLs, which look like this:
> 
> 1226058286.H11540P27364.melampous.ispgateway.de,S=2683

It looks like some system were using Maildir filenames for the UID. That's
legal, but these Maildir filenames change, as they store flags such as
"read" - and IF they change (for instance, if you're accessing the same
mailbox by some other means, or because the server does stupid things),
then there's a problem...

So you'd need to check first if these UIDs are actually constant. I'm not
aware that fetchmail were particularly picky about UIDs.

HTH
