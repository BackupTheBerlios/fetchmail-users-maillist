Dear fetchmail list,

i have a problem with my fetchmail fetching mail from a specific server
with a unique UIDL scheme, using the keep-option.

I'm using the fetchmail binary provided by the Debian etch
distribution (version 6.3.6-1etch1), but tried a self-compiled fetchmail
6.3.9 as well.

The problem is, that fetchmail seems to be unable to remember those
special UIDLs, which look like this:

1226058286.H11540P27364.melampous.ispgateway.de,S=2683

This results in downloading those messages over and over, whenever the
fetchmail daemon is awakened. These UIDLs don't show up in the .fetchids
file (or .fetchmail-UIDL-cache, as it's called in Debian) either.

Other servers work well as expected, with UIDLs like these:

UID2377-1075274572
0MKt64-1H7FUG2K2J-0002jp

Although the UIDLs from this hosting provider seem to be rather
unusual, they don't violate RFC 1939, as far as I can see.

For now, I've temporarily made fetchmail to use imap on these
accounts, but I don't like that and would prefer to use pop3.

Bye,
Burkhard
