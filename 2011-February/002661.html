Jude DaShiell wrote:

> Having read the description of the --silent option I tried using that 
> earlier to find out what it would do.  Another option I tried using was 
> the syslog option to redirect syslog errors to a fetchmail specific file. 
> These were found in the fetchmail man page that came with fetchmail 
> 2.6.3.19 and all I got back from fetchmail was syntax errors.

May I suggest <http://www.fetchmail.info/fetchmail-FAQ.html#G3>?

Note that the rcfile syntax differs from the command line format in that the latter requires "--" before an option and the former requires that these be omitted and also "set" before some global options, and a particular ordering of server-options-before-user-options (per poll).

> It is for 
> that reason I wish that fetchmailconf program would also be able to run in
> command line environments as well as G.U.I. (we don't want to take 
> software away from the teddy bears on wall paper set) 

Feel free to maintain fetchmailconf and port it to nwcurses. 

I plan to abandon it because I don't care to maintain a Tk based tool that is awkward to use, unsuitable for small screens in expert mode (think netbook) and has a user experience that was acceptable two decades ago, but not today.  However, I don't have the time to write/maintain a new tool for the foreseeable future.  The current fetchmailconf does NOT abstract the user interface from its functionality, so porting it to a command line interface would more or less amount to a rewrite.

> because at least 
> that way to the extent fetchmailconf is kept current with working syntax a
> user could run that and sort out incorrect syntax first then if that 
> failed as a result of syntax outside of fetchmailconf's build capacity for
> .fetchmailrc files users of this email list would probably have more 
> intelligent questions to ask.

You're free to ask smart questions already, answering the three crucial points:

- what did you do? (i. e. show the configuration file with made-up passwords, the exact command line)
- what exactly did you get? (i. e. show the error messages/logging verbatim)
- what did you expect instead (unless that's obvious)?

Particular instructions in the FAQ, again, see <http://www.fetchmail.info/fetchmail-FAQ.html#G3>.
