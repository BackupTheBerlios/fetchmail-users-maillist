> fetchmail:/home/jude/.fetchmailrc:3: syntax error at silent

fetchmail is telling you that the syntax of line 3 of your .fetchmailrc
is wrong and in particular the 'silent' statement.

> jude at md:~$ cat .fetchmailrc
> set daemon 300
> set logfile /home/jude/fetchmail.log
> set silent=1

There is no keyword 'silent', only the command mode option '-s' or
'--silent'.  Perhaps there should be.

Also, in the .fetchmailrc the syntax to enable an option is
  'set <keyword>'
and to disable an option
  'set no <keyword>'

> set syslog /home/jude/fetchmail.log

This is incorrect because the 'syslog' option is either on or off, it
does not take a parameter.

If you wish to disable the use of syslog(3) and log everything to a file
then you need to either use the command line options:

  fetchmail --nosyslog --logfile <path/to/logfile>

or have the following statements in your .fetchmailrc file:

  set no syslog
  set logfile <path/to/logfile>

Do _not_ forget to create the log file before you run fetchmail after
adding the above.

I hope the above helps.
