> Okay, the stuff in /usr/share/curl/ca-bundle.crt turned out to be what was 
> needed.  I have gmail downloading with fetchmail but need the syntax to

> order to separate messages that way I can use clamscan to save space when 
> it finds a piece of malware on one of those messages and delete that 
> message.  I had downloaded about 2 CD's worth of unread email and saved it 
> in mbox files and when clamscan found a piece of malware in the mbox file, 
> I couldn't delete just that one message with the malware in it.  So no

Jude,

If you already have

	sslcertck sslcertfile /usr/share/curl/ca-bundle.crt

then adding the fingerprints isn't going to add security, unless there
is a secure way to retrieve the fingerprints from Google.

I doubt that there is a faxback or similar service for gmail
fingerprints though...

Instead, since you can specify only one fingerprint (a fetchmail
limitation in currently released 6.3.X versions), fingerprints may
actually get in the way of mail retrieval -- if it actually does,
depends on the way Google implements server certificates and load balancing.

WRT local mail formats, whether you use mbox, nmh, Maildir, Maildir++,
or whatever, is none of fetchmail's business.
