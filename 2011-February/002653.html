Jonathan,

I've tried to reproduce this - but cannot.  Is libmd found on your system?


I see from the source files how the problem could theoretically manifest 
itself, but the configure[.ac] machinery around avoids that:
Either we include the system md5.h and avoid md5c.c, or we do not 
include the system md5.h and build md5c.c.

- If using NFS or other networked file systems, be sure that client and 
server agree on system time (also check time zones and daylight savings 
time).

- Extract the 6.3.19 tarball into an empty directory, do NOT use a 
config.cache file, and re-run configure and retry. Avoid all shortcuts.

If it still fails, send me your configure, config.log, config.status, 
Makefile.in and Makefile (as bzip2ed, lzma'd, xz'ed or gzip'ed tarball).
