Am 06.02.2011 22:43, schrieb Jude DaShiell:
> I used openssl to show google certificates and extracted those to files
> gmail.crt and google.crt and fetchmail.txt is my .fetchmailrc file and
> fetchmail.out is the failed output from an attempted gmail contact.  Both
> gmail.crt and google.crt are in /etc/ssl/certs/ directory and I've run
> c_rehash /etc/ssl/certs/ repeatedly though neither certificate got picked
> up and installed from what I could tell.  I did the c_rehash operation as
> root too.

Don't do that. You need to download the real root certificate, rather 
than installing the unverified server certificate as trusted. See the 
README.SSL* files of recent fetchmail documents.  Wherever you get the 
gmail root certificate, I don't know, however, most distributors should 
install a ca-certificates or ca-root-certs or similar package which 
would usually be good for Google Mail.  Be sure to use a distribution 
that is still supported by the vendor, not one past end-of-life!
