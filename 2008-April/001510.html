When exactly do you delete the .fetchids file?
Is fetchmail then running in daemon mode?

Fetchmail won't garbage collect UIDs from an unqueried host by itself,
but deleting the "offending" UIDs from .fetchids (or the file
altogether) while fetchmail is currently NOT running should remedy the
situation.

If it doesn't, please show the output of these two command lines:

env LC_ALL=C fetchmail --nosyslog --nodetach --daemon 0 -V
env LC_ALL=C fetchmail --nosyslog --nodetach --daemon 0 -vvv
