Fetchmail functions properly, and apparently it talks to Exchange properly.

#1 Regarding the non-working of the mail:

The material quoted above shows the issue. rt-mailgate cannot establish
trust to the localhost's https web server and tells fetchmail to defer
the message and retry later, which fetchmail does ("not flushed").

How that is all supposed to be set up eludes me, but generally speaking,
either rt-mailgate needs to be told what CA signed the 127.0.0.1 https
server's certificate, and to trust it,
or the 127.0.0.1 https server certificate needs to be installed properly
in the first place.

Review the RT installation instructions regarding certificate
management, web server setup and integration, and thereabouts, and if
you can't get that to work, seek out RT's support channels.

#2 Regarding the length mismatch warning: it's not critical, but should
still be fixed.  For older Exchange versions, see
<http://www.fetchmail.info/fetchmail-FAQ.html#S2> regarding server-side
registry tuning, but I am not sure if that applies to Exchange 2010,
too.  Consult Microsoft's documentation before changing registry, and
make sure to have an up-to-date fresh working backup.
