And, for those not well versed in how *nix works (whether it be unix,
Linux, BSD or whatever), that's pretty normal.  It's very inefficient
to open/write/close a log file every time, so most will open the file
at startup and only do a close/open when explicitly told to do so,
which may simply be a close on exit.  Once a file is open you can
delete it, and this is how some programs handle temporary files.  It
will no longer be visible on disk and you can create a new file with
the same name, but that is a completely different file.  The program
still has the original file open and will happily write to it, filling
up your disk in an invisible manner, until the file is closed.  While
'df' will show the disk filling up, attempts to find the offender will
fail since it's been deleted.
