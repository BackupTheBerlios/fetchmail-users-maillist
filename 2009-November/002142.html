Am 15.11.2009, 17:07 Uhr, schrieb A S <akinstur at yahoo.com>:

> Thanks.
>
> When I have three users in a row like that, under a single poll, is  
> there a way to make fetchmail wait for the first user to finish before  
> going onto the second one, etc.? Otherwise the second and third  
> preconnects throw back bind errors, as ssh is trying to set up three  
> listeners on the same port.

Fetchmail always queries one account after another. In doubt, you can also  
use a "postconnect" command to kill the ssh tunnel before the next account  
will set up its own.

Alternatively, if the destination server allows SSH-preauth, see the  
"plugin" option.

See the manual for details (and let me know if the 6.3.13  
documentation/manpage is unclear; in doubt check  
http://www.fetchmail.info/fetchmail-man.html -- I don't have time to look  
into older manpages).
