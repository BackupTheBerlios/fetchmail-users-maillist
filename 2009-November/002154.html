Rob MacGregor, 2009-11-23 18:44:

> Linux, BSD or whatever), that's pretty normal.  It's very inefficient
> to open/write/close a log file every time, so most will open the file

That might be true for ancient unix flavours, but at least the "very" is
does not apply any more. A quick test shows that on some old P3-800 with
Linux 2.6.x one can easily do more than 100,000 open/close per second.
You don't want your CPU spend all time for that, of course, but almost
all programs do only a few log writes per second, so this is really
nothing of concern, even not for a program like fetchmail.
