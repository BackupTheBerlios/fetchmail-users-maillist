I also employ Dovecot which in turn creates it's file in /var/run which
are then rotated monthly.

/* Snippet */

logfilename [owner:group] mode count size when flags [/pid_file] [sig_num]
==============================================================================

/var/log/dovecot.log root:vmail 0664 1 * $M1D0 CJ /var/run/dovecot/master.pid 30

Dovecot recommends the use of signal 30 when rotating the log. It would
be nice if Fetchmail incorporated some similar feature.

man signal | grep 30

30    SIGUSR1      terminate process    User defined signal 1

It would be a real bonus. Just my 2 cents!
