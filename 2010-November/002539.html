Hi Carlos,

Point taken.  For now, you can sidestep the problem and use fewer "-v" options
to get rid of some of this logging, or use a syslog daemon that can filter out
messages. fetchmail 6.3.19 will fix that and demote this to INFO log.

Currently, fetchmail only uses ERROR (for what otherwise goes to stderr) and
INFO (for what goes to stdout) syslog levels.

I see it would be beneficial to change that in a later release, and it will be
easier to do that given the simplifications for report.c that are already written.
