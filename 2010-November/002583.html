Am 28.11.2010 18:55, schrieb Martin Koeppe:
> 
> Hi all,
> 
> I'm using fetchmail to fetch mails from ISP and forward them to Exchange
> 2007 and 2010 server.
> 
> With that configuration I now saw different cases where fetchmail eats
> the mails, without further notice.

Fetchmail does not do that.

> The problem is when the Exchange server doesn't accept the mail for some
> reason, e.g. mail too big, or client (fetchmail) ip isn't allowed to
> send mail to Exchange, or ...
> 
> In all these cases I would expect fetchmail to leave the message on the
> POP3 server, but it gets flushed nevertheless. As Exchange (in one of
> the cases above, ip address) doesn't accept any mail from fetchmail,
> fetchmail might have tried to send an error mail back, but obviously
> that can't be delivered. So the mail is lost.

So configure your SMTP server to relay bounces from fetchmail.  There is
no way forward unless you can send at least the bounce.

> $  env LC_ALL=C fetchmail -V
> This is fetchmail release 6.3.9+NTLM+SSL+NLS.

Update.  I know there isn't a newer Cygwin package -- but 6.3.9 isn't
supported any more (because it isn't the latest 6.3.X), and you don't
need special patches to make fetchmail work on Cygwin.  It supports
"softbounce" which more or less does what you're asking for -- but it
quite obviously does not fix Exchange misconfigurations.
