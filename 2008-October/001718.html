On Fri, Oct 3, 2008 at 19:11, David A. Cobb <superbiskit at cox.net> wrote:
> Hi!  Please excuse a newbie question.
>
> Ubuntu 8.04.1
>
> I have two mailbox names at my ISP's POP3 server: call them
>     <personal#my.isp.net> and <business at my.isp.net>.  Personal is very
> high volume, because I'm addicted to mailing lists.  Business is very
> low volume -- job search robots and the rare "I saw your resume, and ...".
>
> I cannot get the version of fetchmailconf to run, at present.
> When I duplicated the .fetchmailrc stanza "user "personal" is
> "superbiskit" here, password "mypasswd" ..." to create
> "user "business" is "superbiskit" here, password "anotherone" ..."
> fetchmail retrievs ONLY the second mailbox.
>
> Please advise the appropriate syntax in .fetchmailrc for collecting from
> two boxes on the same server into the same local account.  The ISP will
> not log on the second box while the first is active.

It would have helped if you'd provided your existing config file so we
could work directly with it, however...

poll pop.isp.net
  username myusername
  password mypassword
  is localuser

poll pop.isp.net
  username myotherusername
  password myotherpassword
  is localuser

The second poll line is technically not required, but makes it easier
to read the config file.  Don't forget you can use "fetchmail
--configdump" to see what fetchmail thinks of your config file.
