Anyways, hi Alan,

> I have been using fetchmail for many years to bring my E-mail,
> which accumulates at my ISP, patriot.net, back to my home computer.
>
> Occasionally I travel, and I then use the PatriotNet web site,
> and its SquirrelMail facility, to look at some messages.  These
> messages are then marked as "Read"; and when I get home my
> fetchmail command fails to bring home these "Read" messages.

If this happens with an IMAP account, then I'm sorry, this particular  
feature (UID/UIDVALIDITY) is still missing from fetchmail (and ESR, the  
former maintainer, believed in server-side "Read" tracking - which just  
dropped on your feet).

If this happens with a POP3 account, the uidl option in .fetchmailrc  
should fix it.


> I don't know if it matters, but I run Debian etch, and my
> present fetchmail version is:   fetchmail release 6.3.6+NTLM+SDPS+SSL+NLS

Debian etch's fetchmail 6.3.6 isn't fully patched. Consider upgrading to  
Debian lenny. If you can't: Don't use APOP authentication, don't run  
fetchmail in -v -v mode, and either make sure that your local SMTP server  
accepts all mail or use an MDA.
