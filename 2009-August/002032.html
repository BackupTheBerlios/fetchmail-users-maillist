> Hello Matthias,
>
>> I am announcing the release of fetchmail 6.3.11.  This new stable
>> version of fetchmail fixes a security bug, CVE-2009-2666, and two other
>> issues, see below for details.
>
> due to your security announcement I've translated the new
> binary for the Eisfair (http://www.eisfair.org) community.
> Right after several people have installed it, they reported
> the following error when trying to start the deamon:
>
> *** glibc detected *** double free or corruption (out): 0xbfffb6c0 ***
> fetchmail: terminated with signal 6
>
> One user reported that his was able to start the deamon right
> after he has deactivated all account for which SSL encryption
> is used. Falling back to Fetchmail v6.3.9 solves the problem
> for the moment. Therefore I assume that this error may be caused
> by the newly introduced fix.

The relevant fix information has been forwarded by Torsten Veller; I'd  
suggest to upgrade to 6.3.11 + that patch; and I plan a 6.3.12 release  
real soon. Do not continue to use 6.3.10 or 6.3.9 or older versions.  
Please see my separate regression fix message for more information.

Note that the patch from the security announcement didn't introduce this  
regression, so 6.3.10 + the patch from the security announcement should  
also be safe for the nonce; but that doesn't address two other bugs that  
6.3.11 fixed.

Sorry for the inconvenience!
