Well, at the risk of sounding extremely unknowledgeable about the subject
(but that's only because I am), I can attest that I've thrown lots of

subject. The challenge I've encountered is that every page I've come across

to make sure I'm starting with a correct conceptual overview so as to
minimize my own confusion as I walk through the process.

> 1) What SMTP software you want to use (Sendmail, Postfix, Exim, or others)

So, here's a good example. Do I need to use SMTP software in the first
place? I will not be using SMTP to retrieve mail from my ISP, but rather
want to continue (at least in the near term) to continue to pop it. The
reasons are simple: I believe my ISP blocks inbound port 25 (yes, I'm aware
I can probably substitute another port), and there are anecdotal stories of
my ISP terminating accounts for residential users who run servers
(particularly mail servers). Until I can ascertain the truth of that (and if
it turns out to be untrue, I will want to go the rest of the way and set up
an MX record for my mail server and proceed with receiving SMTP mail), my
plans are to stick to the process that works now and pop mail from them.
Rather than have each mail client be responsible for doing that, though,
through my existing proxy, I want something to take care of it on behalf of
all my mail clients and store the mail centrally on my server. It is my
understanding that Fetchmail will take care of the pop angle; do I need an
SMTP server with which Fetchmail will interact, in order to "store" the
mail, and as well in order to take care of "storing" outbound SMTP mail from
the mail clients (and, for that matter, sending the outbound SMTP mail to my
ISP)?

> 2) What IMAP/POP software you want to use (UWash, Courier, Cyrus or
others)

So, because I want to store mail centrally on my server, I presume I will
need to set up IMAP capability on the server, with which my Outlook mail
clients will interact to view the mail. I've briefly looked at Courier,
which also seems to be a fully-fledged SMTP server as well. That led me to
wonder if I can simply use it in conjunction with Fetchmail to do everything
I want at this point, eliminating the need for something like Postfix (which
I've also looked at).

Again, I apologize for what may seem to be extremely basic questions. Would
I generally be correct in conceptualizing as per the following:

1) Fetchmail will pop my mail from my ISP (on behalf of all my accounts) and
interact with an SMTP server to store it centrally on my server;
2) The SMTP server takes care of properly storing the mail, and of sending
SMTP mail back to my ISP; and
3) The IMAP software I choose will allow my Outlook clients to view and
manipulate the mail stored on my server.

If that's generally correct, then I think I'll take advantage of your
helpfulness and veer perhaps wildly off-topic to ask about calendaring,
contacts, and all the other related stuff one normally associates with an
Outlook/Exchange environment. Does any of the software we're discussing
handle these as well, or in my scenario would I still have a PST local to
each client workstation that would store these items, leaving only the mail
on the central server?
