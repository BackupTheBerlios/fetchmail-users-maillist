Rob MacGregor wrote:
> On Mon, Aug 31, 2009 at 02:40, Dave Beach<drbeach at rogers.com> wrote:
> <---SNIP--->
>> If I understand fetchmail correctly, it will allow me to regularly poll my
>> ISP and pop mail for all of my e-mail accounts, and interact with mail
>> server software on my server to place mail in the appropriate directories. I
>> presumably can then run other software to allow the client Outlook
>> installations to access the server-based e-mail using IMAP. I presume I can
>> rely on some sort of MTA functionality on my server to continue to send
>> outbound SMTP mail to my ISP.
> 
> Sort of.  Fetchmail allows you to retrieve email from your ISP (and
> any other POP/IMAP servers) and pass it to your local SMTP server.  It
> is up to your SMTP server what happens next.
> 
>> With that as a summary of what I'd like to do, I would be very grateful if
>> someone would conceptually walk me through how this might work, and what
>> software would be applicable at what stages. I'm not after a detailed
>> explanation of how to configure the software, but rather a general overview
>> of how it would all work together and what role each software would play. I
>> am a complete newbie when it comes to implementing and using mail server
>> software.
> 
> There is no shortage of information out there on this subject.  Throw
> appropriate words into any search engine and you'll get a host of
> pages taking you through the process.  You'll need to make 2
> decisions:
> 
> 1) What SMTP software you want to use (Sendmail, Postfix, Exim, or others)
> 2) What IMAP/POP software you want to use (UWash, Courier, Cyrus or others)
> 
> There's no single right answer here.  It all depends on what size of
> installation you've got and what features you want.  You'll find
> people who'll argue strongly for/against particular software because
> of the author, or some historical event relating to the software.  I'd
> recommend you make your own decisions ;)  You may also want to look at
> one of the all-in-one packages that are available to make your life
> easier.
> 
> There's nothing particularly difficult about what you're planning -
> it's the setup I've got myself.
> 
I can second Rob.
I use the same configuration with fetchmail obtaining the mail from my 
provider, Postfix as MTA amd Dovecot as POP/IMAP distributor for over 10 
years (although I have also used sendmail and qmail as MTA's but Postfix 
  suits me best). But any combination should work.
