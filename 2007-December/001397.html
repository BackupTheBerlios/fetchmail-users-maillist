> Hi,
>
> after revisiting my fetchmail logfile, I discovered the following at the
> end of each poll:
>
> 	Invalid SSL protocol '' specified, using default (SSLv23).
>
> However, all seems to work flawlessly. I did never see this before,
> and nothing has changed here since many months. I'm quite clueless what
> could be the cause. Setting sslproto doesn't work either.

It's caused by the failed CAPA probe:

> fetchmail: pop.online.no fingerprints match.
> fetchmail: POP3< +OK <22177.1197310179 at mail54.nsc.no>
> fetchmail: POP3> CAPA
> fetchmail: POP3< -ERR authorization first
> fetchmail: authorization first
> fetchmail: Repoll immediately on xxxxxx at online.no@pop.online.no
...
> Invalid SSL protocol '' specified, using default (SSLv23).

The fix will be in fetchmail-6.3.9, in the meanwhile, you can apply the
attached patch.
