Sorry this phrase is unintelligible. You do indeed need a way to let
fetchmail snatch the original envelope recipient header from the
messages it downloads. If your IMAP or POP3 server writes

> With my current fetchmailrc config file (see below), all the mail is
> being sent to the one citadel account, despite having set up another
> user who I have tried sending mail to. That user receives nothing atm.

Try running fetchmail --nosyslog --nodetach -v -v (perhaps a third -v if
needed) and see if it matches the domain. If not, some "aka
other.doma.in" may help. In your situation, fetchmail will only
recognize

X-Envelope-To: SOMELOCALUSER at mailhost.zen.co.uk

(with varying SOMELOCALUSER parts, the rest is matched verbatim).
