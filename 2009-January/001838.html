> > Also, can I safely run fetchmail (in keep mode) whilst the fetchmail
> > daemon is still running in the background, both accessing the same
> > mailbox?
> 
> You shouldn't do that.

It would also require some trickery, since fetchmail usually locks its
home directory (equal to the invoking user's home directory, unless
overridden by environment variables) and prevents being started for a
second time; it will instead just signal the background daemon to start
a new poll cycle.

Many POP3 servers will also lock the mailbox if one client is active, so
that a second cannot access while the first one is live.
