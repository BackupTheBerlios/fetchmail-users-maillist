I wrote:

> I have uploaded the third fetchmail 6.3.8 release candidate to the usual
> download location: <http://home.pages.de/~mandree/fetchmail/>.

NOTE that fetchmail 6.3.8 is the last planned fetchmail 6.3.X release.
I plan to put the 6.3 branch on hold and only fix severe bugs
afterwards, so as to free resources for spawning a new branch.

The new branch will make some more radical changes to the code, remove
legacy code, misguided attempts, and clean up the mess - perhaps we'll
see a few features as well.

So, if you want your bug to be fixed before the next release, it is very
important that you test 6.3.8-rc3 and report issues quickly, I plan to
release 6.3.8 in 8 - 10 days time.


I don't expect the new branch to make headway quickly though, I'm busy
with real-time work, the only active fetchmail developer at this
time, and living in the northern hemisphere means spring and summer are
approaching, so I'll probably spend less time on coding anyways...
