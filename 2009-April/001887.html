Hi folks,

I've been successfully using Fetchmail for about a year to pull mail 
off Gmail accounts and dump it into a Mailman mailing list system. 
The Gmail antispam works about twice or three times as well as 
anything I've been able to put on a local server, without tying up my 
bandwidth.

I'm in the process of building a new server.  Mailman is working fine 
(I can send mail directly to an alternate list address and it passes 
through properly), but I can't seem to get Fetchmail to do anything.

Nothing writes to fetchmail.log.  Nothing is returned with

	fetchmail  --fetchmailrc /etc/fetchmailrc -v

  (or -v -v, or -v -v -v)
or

  	fetchmail  --fetchmailrc /etc/fetchmailrc -k -m cat


My fetchmailrc is of the form:

poll pop.gmail.com with protocol POP3 port 995
username "eriksgigs at domain.com" there with password "xxxxx" is "mailman" here
and options ssl sslcertck sslcertpath /etc/ssl/certs
mda "/opt/local/share/mailman/mail/mailman post eriksgigs" set daemon 900


I *think* I have my certs set properly, having copied the equifax and 
googlepop certs from my working install and having followed the 
instructions in the FAQ, including the rehashing.  I just double 
checked it all.

The working server is running Mac OSX 10.3.9.  The new one is on 
10.5.6 (both "client" version).

Where do I look next?
