Hello,

I have been trying to set up fetchmail as an imap to smtp gateway to deliver voicemail messages from a cisco callmanager business edition to exchange for a customer and have run into two problems.  The first is that when I don't use the fetchall option the imap server bombs out on the search command.

fetchmail: IMAP> A0006 SEARCH UNSEEN NOT DELETED
fetchmail: IMAP< * BAD The command is not valid in the current state

I've looked around (even read RFC3501) and can't find the reason for this error.  Using fetchall work fine but if anything changes on the imap server the users get all of their old messages delivered again.  Does anyone know why this happens?

The second issue has to do with the voicemail message as an attachment.  The users can see it and play it in Outlook but not in Outlook Web Access - paperclip shows there is an attachment but it does not show up when opening the email.  This seems to be an issue with Exchange (customer has 2003 SP2).  I found a hotfix at microsoft but it only applies to the RFM edition.  I tried the pass8bits, stripcr, forcecr, etc. options but nothing worked (it's obviously an issue with the encoding of the attachments as far as I can tell).  Does anyone have a work around for this issue other than it's a microsoft "feature"?
