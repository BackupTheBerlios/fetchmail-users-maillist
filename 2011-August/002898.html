If you can verify that self-signed cert' is authentic (because there's a
fingerprint, or at least a secure and verified https:// channel for the
download where you didn't need to allow strange certificates, IOW there
is no man-in-the-middle attack going on), you can mark that as trusted.
 This declaration is made by just putting the certificate into the
trusted store, for instance, /etc/ssl/certs (system-wide) or any
directory that you point --sslcertfile or --sslcerpath to (which see in
the manual, only available in some of the newer fetchmail versions ;-))

Note that if using sslcertpath-style entries (i. e. a directory with
certficiates), you need to run c_rehash on that directory so you get the
relevant abcdef12.0 symlinks that OpenSSL needs. Otherwise, just
concatenate the PEM-format certificates together and use --sslcertfile.
