The asterisks (stars) don't belong there.

For reliability, add the "uidl" option after POP3.

> downloaded the gmail certificates
> $ openssl s_client -connect pop.gmail.com:995 -showcerts
> saved one certificate as gmail.pem
> one as equifax.pem

Don't do that, you evidently don't know what you're doing, and whatever
website proposed to do that, should be taken down.

Where have you read this procedure?


Solution:

Install and configure the ca-certificates Ubuntu package (through
debconf and select the certs you want to trust) and remove the
sslcertpath option from fetchmail's configuration, fetchmail will then
use OpenSSL's default, and it should work.

> what went wrong? I've checked the fetchmail archive and not come up with a
> solution to this.

The solution is to download the Equifax certificate (that Google isn't
presenting to clients, because clients need it installed in the trust
store anyways) from a *secure* and *verified* location, check the finger
print, and only then install it.

The ca-certificates package is the easier sidestep though - there, the
Ubuntu packager is supposed to have verified the certs already.
