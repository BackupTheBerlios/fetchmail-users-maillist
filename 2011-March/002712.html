IMO this sort of capability does not really belong in fetchmail,
which -- being a Unix tool -- is supposed to do one thing well.
Fetchmail's job, which it does very well, is to retrieve mail
from mailboxes and pass it to another local mail agent for further
processing (often local delivery).

The solution to the OP's problem is to configure a local SMTP
forwarder -- another tool which does one job well -- to receive
the mail from fetchmail and pass it to the Exchange server via
TLS, queueing it locally if necessary (e.g. when the Exchange
server is down).  This approach will avoid the problem of having
to repeatedly retrieve the same mail from the POP3 server due to
the Exchange server being unavailable for one reason or another.
