Am 08.03.2011 21:13, schrieb robbticb at wp.pl:
> how can I forward mails to Exchange 2010 with TLS over Port 587?
> 
> receiving mails from POP3 is woking fine.
> 
> .fetchmailrc
> 
> defaults
>     proto pop3
> poll mail.mydomain.com
>     proto pop3
>     auth password
>     user "user at mydomain.com"
>     pass "my-password"
>     is user-local
>     ssl
> 
> And now forwarding to Exchange (IP=192.168.1.10)
> 
>     smtphost 192.168.1.10/587 smtpname exchange-user at 192.168.1.10
> 
> Now I get following error from fetchmail:
> fetchmail: SMTP error: 530 5.7.1 Client was not authenticated
> SMTP server requires STARTTLS, keeping message.
> 
> how can I set Password and TLS for SMTP forwarding?

Unfortunately, current fetchmail releases do not support SSL-wrapped or
STARTTLS-enabled SMTP.  So you need to work around that.

One option is to run an MTA on the local computer that accepts mail and
sends to exchange (options are for instance, but not limited to, Postfix
and Exim, quite possibly also Sendmail).

Another option is to install msmtp <http://msmtp.sourceforge.net/> and
run it with the "mda" option.
Note that this is prone to creating mail loops unless configured
properly (you may want to use "exchange-user@[192.168.1.10]" rather than
%T on the command line to avoid that risk) and note that I haven't tried
that.  Be sure to carefully read the fetchmail manpage WRT the --mda
option for information on how to (not) quote arguments and variables.
