> 
>> how can I forward mails to Exchange 2010 with TLS over Port 587?
>>
>> receiving mails from POP3 is woking fine.
>>
>> .fetchmailrc
>>
>> defaults
>>      proto pop3
>> poll mail.mydomain.com
>>      proto pop3
>>      auth password
>>      user "user at mydomain.com"
>>      pass "my-password"
>>      is user-local
>>      ssl
>>
>> And now forwarding to Exchange (IP=192.168.1.10)
>>
>>      smtphost 192.168.1.10/587 smtpname exchange-user at 192.168.1.10
>>
>> Now I get following error from fetchmail:
>> fetchmail: SMTP error: 530 5.7.1 Client was not authenticated
>> SMTP server requires STARTTLS, keeping message.
>>
>> how can I set Password and TLS for SMTP forwarding
> 
> I may be wrong; however, I had a similar problem with forwarding to a
> Postfix machine. Fetchmail does not offer SMTP Authentication or TLS
> from what I have been told. I was forced to configure the Postfix
> machine to accept on a non default port number and not require SMTP
> Authentication or TLS.

It does support SMTP authentication (look for options starting with
esmtp in the documentation) but not currently (as of 6.3.19) SSL,
STARTTLS, or TLS for SMTP or LMTP.

> No matter how you look at it, it sucks.

It may suck, but this is Unix, so you can work around fetchmail's
limitation with msmtp -- as documented in my earlier message in this thread.
