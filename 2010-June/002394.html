Am 25.06.2010, 00:06 Uhr, schrieb Lorenzo Milesi:

> Hi.
>
> I configured a system fetchmail to use expunge, in order to be sure I  
> could delete mails.
> Sadly, this doesn't seem to work...
> My /etc/fetchmailrc looks as follows:
>
> set daemon 300
> set syslog
> defaults proto pop3 timeout 18000 uidl expunge 500
> poll my.mail.server proto pop3
>     user "my_account" password "my_pass" is "user"  keep expunge 50
>
> but I see no command after the 50th deleted mail. It goes straight to  
> the end, when the pop3 server takes ages to respond to QUIT (I have a  
> lot of messages on this mbox), so wasting time.
> I tried several times but no way.
>
> What's wrong?

The configuration is contradictory. "keep" prevents deletions altogether -  
and then there will be nothing to expunge.

> using fm 6.3.6-1etch2 on debian etch.

This is an ancient fetchmail version, three and a half years old, there  
have been 11 bugfix releases since to date. 6.3.6 is unsupported.  You  
were lucky this time because it appears to be a configuration issue.

If a reconfiguration does not solve it, please build 6.3.17 from source  
and run it according to <http://www.fetchmail.info/fetchmail-FAQ.html#G3>.

Debian/Etch also has been out of support support since February 2010.
Please upgrade to Debian/Lenny.
