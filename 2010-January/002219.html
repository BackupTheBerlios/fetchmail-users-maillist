1) certificates, etc. nicely configured

2) $ openssl s_client -connect pop.gmail.com:995 -CApath ~/.certs/
indicates that all is in order:
                +OK Gpop ready for requests from 74.181.252.241 
5pf28250123ywh.11

.fetchmailrc file looks like this:
set postmaster "mylocalusername"
set daemon 600
poll pop.gmail.com with proto POP3

mylocalusername here options ssl
    sslcertck sslcertpath /home/robert/.certs/

3) $ fetchmail -d0 -vK pop.gmail.com
