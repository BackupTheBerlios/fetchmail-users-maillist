>> 3. Does fetchmail re-probe capabilities in daemon mode for the 2nd and
>> subsequent polls? It shouldn't.
>
> What if fetchmail runs for 3 months, and the server changed 2.5 months
> ago? And with *ix, 3 months is nothing.

In that case, the way out is to have fetchmail re-execute itself.  To
achieve that, just touch the run control file ($HOME/.fetchmailrc, for
instance) - if the mtime increases, fetchmail restarts.
