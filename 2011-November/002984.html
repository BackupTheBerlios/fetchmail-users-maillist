Probably because you forgot to provide the full path to fetchmail.

> I've also tried using Webmin to check the box "Start at boot" but it doesn't take.  The box won't stay checked!?.
>
> When I run the same command above from a terminal as root it tells me fetchmailrc must be owned by me.  It's not.  It's owned by fetchmail.
>
> So I set up a duplicate copy in /root and owned by root.  Then,
> fetchmail -d 60 -f /root/fetchmailrc
> works ok.  But from a console.
>
> So how do I get it to start at boot?
>
> This is Ubuntu 11.04.

The simplest thing to do is to use Ubuntu's package manager to install
Fetchmail. Then you'll get the startup script and can use the built in
startup management tools (such as "update-rc.d"). It does mean you'll
likely be running an out of date version of Fetchmail, so if you
encounter problems you may need to:

1. Back up the startup script
2. Remove the package
3. Install Fetchmail from source
4. Restore the startup script, changing the path to fetchmail to suit
your install
