Host does not use getaddrinfo() or similar from the libc's resolver
interface, but does its own DNS lookup, using BIND 9 libraries, so it
does not match what fetchmail is doing.

Check your nsswitch.conf (where present, f. i. on Linux, Solaris,
FreeBSD), host.conf, hosts, resolv.conf, and possibly network and router
configuration for rogue DNS resolvers ("servers").  Also check with your
router's vendor if there are any firmware updates, and be sure that
nobody has modified the router's or your computer's configuration.


Explicit queries go like this:

dig @dns.ser.vers.ip pop.gmx.net. aaaa
dig @dns.ser.vers.ip pop.gmx.net. a

for instance

dig @192.168.0.1 pop.gmx.net. aaaa

which comes up empty. Hence no IPv6 address.

And a yields two IPv4 addresses.
