The problem occurs when your domain has a * record and you added the 
domain as a search domain in /etc/resolv.conf. When you do now a lookup 
for a server that has no AAAA record you will get the A records with the 
correct values but additionally a lookup will be made for the host plus 
your search domain. That will then consequently match your * record and 
your own server is found as well, just as it happens for pop.gmx.net.
It's almost what is described here: 
http://serverfault.com/questions/383266/search-and-domain-in-resolv-conf-slow-lookups-on-ubuntu 

Once I remove the search domain things work as I would like them to do. 
What would be nice would a optional similar to ndots 
(http://manpages.ubuntu.com/manpages/lucid/man5/resolv.conf.5.html) that 
allows you to only append the search domain if the given hostname has 
more then n dots or matches some expression or so.
So for now the best solution seems to be to either not have a * reconrd 
for your domain or to not have a search domain configured.
