Hi Gerard,

your report lacks essential information, see
http://www.fetchmail.info/fetchmail-FAQ.html#G3

The bit of information you quoted suggests that the message fetchmail tried to
retrieve did not have a proper separating line between header and body (or
undetectable through malformatting) and additionally ended before a blank line.

Looking into this, there is a corner case inside fetchmail where it would
complain that way about a message that is actually RFC 5322 compliant, but in
that case, fetchmail wouldn't write the "incorrect header line found" log line -
so this bug isn't triggered in your case.

Please provide me with a verbose mode trace as suggested in the URL mentioned
above so I can see what's going on.
