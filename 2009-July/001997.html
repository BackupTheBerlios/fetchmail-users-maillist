On Sun, Jul 26, 2009 at 08:39, Heinz Diehl<htd at fancy-poultry.org> wrote:
> Hi,
>
> I'm using fetchmail 6.3.9 on Linux since many years.
> Today, I encountered the problem no longer to be able to get mail from my
> providers pop3 server. It worked a long time before, and I didn't change
> anything.
>
> The problem is that fetchmail tries to connect to the ip
> "1.0.0.0" which it claims should be "pop.online.no", and fails, of course.

Then you have a name resolution problem, which is nothing to do with fetchmail.

Check your hosts file (/etc/hosts) and what::

nslookup pop.online.no

returns.  If it returns 1.0.0.0 then the problem is with your DNS
server (see /etc/resolv.conf).
