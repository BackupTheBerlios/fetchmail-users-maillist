That should not be the cause of the problem, name resolution happens in  
user space, not inside the kernel.


The other issue I have is that nslookup isn't exactly a useful debugging  
tool, it has somewhat arcane implications about its name servers; "dig" is  
much more useful, so can you show the output of several commands:

- cat /etc/resolv.conf

- dig pop.online.no a

and for any "nameserver" line you find in /etc/resolv.conf, replacing  
1.2.3.4 by the nameserver's IP address:

- dig @1.2.3.4 pop.online.no any

and as root:

- lsof -n -P -i:53


I see several possible causes:

- hardware failure (if other applications crash randomly or behave in  
strange ways, this may be it)

- Your name server is broken - it may be your ISP's one.
