You're making up log information. Don't do that.
The UIDLs betray the identity of the server anyways. 8-)

Issuer Organization/Server CommonName however are hints that fetchmail uses SSL
or TLS.

> fetchmail: POP3> USER "USER"
> fetchmail: POP3< +OK
> fetchmail: POP3> PASS *
> fetchmail: POP3< +OK Logged in.

> now my questions
> 
> 1) how can i see/check that i am using ssl?

see above.

> 2) does the content of line "fetchmail: POP3< SASL PLAIN LOGIN" means  
> that i am using plain text for authentication?

You are using plain text authentication through a secure SSL-wrapped channel.

However, you can tell that from the USER/PASS lines. SASL PLAIN LOGIN is the
advertisement of the server and not what fetchmail has chosen. See RFC 1939 and
RFC 2449.

> 3) how can i see which certificate was used/tried for a connection?

You can't. Either the proper root certificate for the chain is in the
.../ssl/certs/ directory (it is in your case, since you've used ssl and
sslcertck and fetchmail has attempted to log in after the certificates were
verified OK), then fetchmail/openSSL will trust all certificate chains signed by
the respective root certificate, or it's not.

> i guess i configured ssl but i am not using it.

Why do you guess that?
What needs to be improved in logging to make you believe you're using SSL/TLS?
