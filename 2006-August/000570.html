On 8/19/06, Bob Meier <orcomza at gmail.com> wrote:
> Hi:
>
>  I'm using Debian Sarge with fetchmail version 6.2.5-12sarge4.

You should be on 6.3.4

>  How may setup fetchmail to download mail at different intervals for some
> users?

Provide different interval options:

poll pop.isp1.net user first password firspw interval 1
poll pop.isp2.com user second password secondpw interval 2

That would poll pop.isp1.net every run and pop.isp2.com every second.

>  I didn't understand if I can mix system wide fetchmailrc with private or
> personal .fetchmailrcs for _some_ users.

Yes, you can though you don't have to.

>  If I create .fetchmailrcs, which process runs it? Do I have to create cron
> jobs for this users?

You would need to either run a daemon as those users or use cron to
schedule the runs.
