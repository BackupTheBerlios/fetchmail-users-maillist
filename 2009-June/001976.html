> Now everything is working but I still got the authentication error.

So password and/or login are wrong, or the server doesn't let you login  
without SSL.

> poll pop.neuroweave.nl with proto POP3 envelope X-Envelope-Rcpt
>
>      aka neuroweave.nl localdomains nwhq.nl
>         user 'jlblom at neuroweave.nl' there with password ......options  
> forcecr
> 	sslcertck
> 	sslcertpath /home/joep/.certs
> #	ssl

Note that this uses opportunistic TLS, i. e. no encryption if the server  
doesn't offer it. Either uncomment the ssl, or put "sslproto tls1" there  
if you need STARTTLS (sorry for the misnomer).

>   Do I maybe have to repeat the ssl commands for the other user?

Yes, you do.

Always try verbose mode when trying to debug.
