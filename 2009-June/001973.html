> Hi,
> I use fetchmail since many years to fetch my mail from my mail-server
> which pointed to my domain neuroweave.nl.
> Due to problems with the hosting provider, the domain host changed
> several times (automatically) and the domain remained visible.
> However, 2,5 weeks ago without my knowledge ( due to an "administrative
> error!") my domain was removed.
> To make a long story short I now have de domain hosted again on a server
> which, however, has no public certficate.
> Now fetchmail complains, as follows:
>
> fetchmail: awakened at Wed 17 Jun 2009 04:11:56 PM CEST
> fetchmail: Server certificate verification error: self signed certificate
> fetchmail: Authorization failure on jlblom at VhostID2.mcnolia.nl
> fetchmail: Query status=3 (AUTHFAIL)
> fetchmail: sleeping at Wed 17 Jun 2009 04:12:01 PM CEST for 600 seconds
> My .fetchmailrc file is:
> .....
> poll VhostID2.mcnolia.nl with proto POP3 envelope X-Envelope-Rcpt
> #    aka neuroweave.nl localdomains nwhq.nl
>         user 'jlblom' there with password 'xxxxxx' is 'joep' 'jlblom'
> 'music' 'info' 'jl.blom' 'joep&lon' 'composer' here options forcecr
> 	sslproto tls1
>
> The aka in or out has no effect.
> Why bother with a certificate? I have the sslproto option used but it
> has no effect.
> I would like to have a  nosslcertck option to override this check or is
> there another possibility?

sslcertck is off by default, so unless you enable it in another place of  
the fetchmailrc file that you haven't shown (defaults section?), or on the  
command line, this error does not cause the authorization failure - it's  
more likely that the credentials are refused by the
server.

Please try verbose mode and see if the logging gets clearer - for details,  
please see <http://www.fetchmail.info/fetchmail-FAQ.html#G3>
