There was some discussion on this list regarding configuring Fetchmail
with Postfix's 'Virtual' deliver (Postfix virtual MAILBOX example:
separate domains, non-UNIX accounts) -
<http://www.postfix.org/VIRTUAL_README.html> I am using the following
in my .fetchmailrc file and it works fine.

This is an example for 'GMail'. I have obviously substituted 'me' for
the actual user and 'virtual-domain' for the true domain name. Your
'sslcertpath' will differ. It is vital that your use the 'dropdelivered'
option or else mail will loop when using Postfix.

poll pop.gmail.com with proto POP3 timeout 30 and options uidl
   user 'recent:me at gmail.com' there with password 'secret' is 'me' here
   options forcecr dropdelivered
   smtpaddress 'virtual-domain.net' ssl
   sslcertpath '/usr/local/etc/postfix/certs/'
   sslfingerprint '92:73:17:4C:34:4B:68:F7:B2:17:71:42:0D:7F:9F:33'
