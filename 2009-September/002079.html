>
> I have my mail on an IMAP server.  I want to keep it there, for the most
> part, but I have a limited amount of space, so now I want to grab my
> oldest mail off IMAP and archive it to a local drive somewhere.
>
> Figuring out which mail is easy, I've already done it by putting all the
> mail I want to archive into a particular IMAP folder.
>
> Now I was hoping to use fetchmail to get all this mail off this one IMAP
> folder into a local mail store that I can then gzip and stuff away as
> an archive.  I imagine that fetchmail can do this, but it seems my
> Google-fu is failing me.  Can anyone provide some pointers on how to
> set up a configuration for this?

I'd start by reading the man page - it contains everything you need ;)

For selecting a folder other than the INBOX you want the "folder"
keyword.  As for creating a local archive, that's down to the SMTP
server you have fetchmail hand the mail over to (or the MDA).
