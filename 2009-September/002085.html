Gerard schrieb:
> Can fetchmail be used in conjunction with Postfix virtual MAILBOX
> separate domains, non-UNIX accounts.

I take it you mean fetchmail forwards into Postfix's virtual(8) delivery agent,
rather than fetching from one.

Fetchmail itself does not depend on user accounts except for the one that
fetchmail runs as (i. e. there should be a dedicated, unprivileged account that
you use to start fetchmail).

Whatever the destination server makes of the addresses is outside of fetchmail's
scope, it just passes addresses on, possibly with some rewriting if you use user
mappings.

> If anyone here has it actually
> working, ere there any problems in configuring it? The mail would be
> delivered directly to Postfix and not an MDA like procmail.

I'm not aware of configuration difficulties here. For multidrop, be sure to set
the proper header with the "envelope" option.

Avoiding procmail is the right thing to do, in your particular case, and
generally, too.
