> The process for inbound email, in general is:
>
> Sender -> Their ISP's SMTP -> Your ISP's SMTP -> Your ISP's POP/IMAP
>
> When you add fetchmail this adds the following to the end of that:
>
> -> fetchmail -> Your SMTP -> Your POP/IMAP
>
> Fetchmail simply collects the email from a remote POP/IMAP host and
> then hands it over to another SMTP server (MTA).
>
>> Okay, but mail has to be stored somewhere, right, by something?
>
> By your POP/IMAP server.

Well - actually the MTA (SMTP server) (Postfix, Sendmail, Exim, Courier)  
-- stores mail that arrives initially (no matter how). To be even more  
precise, the MDA does. It is either an integral part of the MTA (Postfix's  
local and virtual are MDAs, Exim itself represents one) or a separate  
software (the discouraged procmail or the better maildrop).

>> Okay, then perhaps I really have a terminology problem here. As above,

>> been using "mail server" as synonymous with "SMTP server" or "MTA". It
>> sounds like I need to think of the IMAP software (courier-imap, etc) as  
>> the
>> "mail server". Am I more correct now?
>
> Maybe ;)

Well - it depends on how you define "server". If it's the tangible machine  
in itself or one abstractly one particular piece of software that you're  
running. The first can comprise several of the latter.
