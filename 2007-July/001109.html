Fetchmail believes it has just deleted three messages and kept the other
seen messages. This UID stuff keeps some history, if you've used different
options (such as --keep) previously that might explain why those messages
are there.

Check the manpage for the --all and --flush options. Use the former to
retrieve all messages again (possibly delivering some twice) - you should
only need this once - or the latter to flush all seen messages without
retrieving them. --flush is more dangerous, too.
