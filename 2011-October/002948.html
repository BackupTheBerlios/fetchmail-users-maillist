There are development versions of fetchmail that use Exchanges's MAPI -
they need a fairly recent OpenChange (libmapi) version to build, and I
haven't tested them for lack of access to an Exchange server.  Chances
are they don't work at all.

If you want to help testing, and you're familiar with building
development versions of software and use enough of Git to clone/update a
repository and switch branches, grab the BRANCH_MAPI branch from
fetchmail's Git repository at http://gitorious.org/fetchmail/ - i. e.
clone with Git, and however, I cannot currently offer any support with
how you do that beyond what follows (others on the list are free to
help, but I'm not sure if anyone will), and what's in README.mapi.

All requisites being installed, the procedure would be:

autoreconf -ifv
./configure --enable-MAPI --with-ssl
make check
make install

And then configure along README.mapi.

Note that MAPI doesn't use SSL or TLS (other protocols might) - but
since you're using SSH, that shouldn't be an issue.
