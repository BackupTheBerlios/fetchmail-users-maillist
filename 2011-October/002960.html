This was set up using Macports, which installs most of Fetchmail in
/opt/local.  As you can see from the first stuff I sent, fetchmailrc is at
/etc/fetchmailrc, and it's obviously being read.

The purpose of this install is to pull down mail intended for a series of
Mailman discussion lists from Gmail, which we use to run our domain name's
mail.  Fetchmail hands the incoming mail to Mailman, which then processes it
and sends it out to the appropriate discussion list. Therefore Fetchmail
does not need an outgoing mailserver set up.
