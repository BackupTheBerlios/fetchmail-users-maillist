Ok, here it is:

set syslog DOES send status report to /var/log/mail.log, file cannot be read by
normal user, this setting cannot be overriden by command line option
--nodetach

set logfile MUST be given the full path to logfile
(/home/dave/.fetchmaillog) NOT ~/.fetchmaillog or the .fetchmailrc line
'set logfile' is ignored and status is printed to screen.  This option CAN
be overridden by the option --nodetach

'set syslog' line before 'set logfile' line in .fetchmailrc has the effect
of using 'set syslog' first, the second option ignored... (as most .rc files
are executed top to bottom in common usage)
