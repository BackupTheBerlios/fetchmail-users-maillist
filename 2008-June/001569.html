Hey Fetchmail list,

I'm working on a large POP3 mail migration project (UW to Dovecot) where it's
become critical to preserve the UIDLs from the old POP server for use with the
new.  I can do this if I handle the migration using files, but considering how
many mailboxes and how much mail is involved the downtime required to conduct
the migration becomes a problem.

I'm hoping to use fetchmail to grab mail from the old mail server and have it
deliver on the new mail server.  This way I can start the migration while the
old server is still in production, preserving mail and keeping track of UIDLs
(fetchmail -p pop3 -k -U) and then during a short downtime do a final fetch and
cut over.

The problem is that all the UIDLs will change and all the pop messages will 
appear as new on the new dovecot server.  If I could add the X-UIDL: header to 
the messages when fetchmail grabs them then I could set Dovecot to honor the 
UIDL listed in the message and the problem would be avoided.

Is there a fetchmail patch out there that does this?

Any clues would be appreciated, thank you.
