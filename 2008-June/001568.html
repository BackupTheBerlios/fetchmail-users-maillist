> I run my regular fetchmail for my "standard" mailboxes  all  15  minutes
> but since I have installed a seperated tool which require fetchmail  and
> must run all 5 minutes I am running into trouble.
> 
> Since all two  fetchmail  instances  run  from  a  script,  how  must  I
> manipulate it, that I can run more then one instance of fetchmail?
> 
> OK, it is already enough, if the script of my second fetchmail  instance
> must be changed since the  "normal"  fetchmail  process  should  not  be
> touched.

If you specify all the options on the command line, just make sure there
are different values for the --pidfile and, possibly, the --idfile
parameter.  The --idfile may be important, since there might be trouble
if two instances of fetchmail try to update the fetchids file at
the same time, though I haven't checked if this is taken care of by
some locking.

My advice would be to create separate fetchmailrc files.  In those, make
sure you specify different values in the "set idfile" statement, and
you will still need to give a --pidfile command-line option to one of
the fetchmail instances.

Hope that helps... although I may not have worded it too clearly.
