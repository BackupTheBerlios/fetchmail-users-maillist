>> Hi, doing some clean up I noticed that the fetchmail user on my system is
>> configged for /bin/sh ratehr than /bin/false or something else. Can I change
>> this to /bin/false or does the user require execution permisions?

Depends on how you're using (and particularly starting) it, and how your
system handles such passwd entries.

The passwd database usually lists login shells, whereas the system
interfaces that fetchmail uses for mda/plugin/plugout would hardcode the
use of "sh" and should be unaffected, at least on POSIX compliant
systems (unless I have missed something).

The exact impact is apparently system dependent, and also installation
dependent, so I'd say the quickest way to find out is testing it,
possibly with "-all --keep" options; and options that run commands are
candidates for failure, such as mda, plugin, plugout.  If you do not
need external commands through these options, and your system can still
start fetchmail if the shell is /bin/false or /sbin/nologin or
thereabouts, it should be safe to change. Be sure to have at least one
message delivery attempt, else the test isn't useful.
