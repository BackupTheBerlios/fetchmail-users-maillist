I think, it would be easy to test the issue of buffering by using
redirection. Just run fetchmail twice remotely, once with redirection:

rsh ... fetchmail
rsh ... 'fetchmail 2>&1'

If the output is unchanged, buffering is not the issue. If the output
changes, then stdout is getting buffered.
