On Sat, Apr 23, 2011 at 11:38, Jerry <fetchmail.user at seibercom.net> wrote:
> I am not even sure if this is possible; however, since I was asked to
> look into the feasibility of it I thought I might as well try.
>
> The system is a FreeBSD 8.2-STABLE amd64 running Postfix
> (2.9-20110323), Dovecot-1.2.16 and the latest version of Fetchmail.
>
> Scenario:
>
> All of the users are hosted on a Google App's account. The user wants
> to use one account in particular with "plus addressing"; ie,
> <joe+sales at domain.com> for example. Actually, there would be several
> "plus" combinations, although most would involve only the one e-mail
> account. The idea is to fetch mail from the Google App account, forward
> it to Postfix and onto Dovecot where dovecot would invoke sieve to
> filter the mail.
>
> We have tried all of the common examples shown in various web examples;
> however, we have not found a way to make this work. Presently, we have
> not found a way to get the envelope address and forward it Postfix
> which in turn is configured to send it onto Dovecot. Mail sent directly
> to this server with "plus" addressing works correctly so I am confident
> that we have that portion configured correctly.
>
> If someone could supply us with a working solution it would be most
> appreciated. I have just not been able to come up with anything that
> works consistently.

A simple multidrop configuration should handle that. Configure
fetchmail to use the "Delivered-To:" header for the Envelope and it
should just work - certainly it has for me.
