On Thu, 21 Apr 2011, Matthias Andree wrote:
> Am 18.04.2011 14:41, schrieb Martin Koeppe:
>> I encountered a configuration problem with fetchmail.
>>
>> $ fetchmail -V
>> This is fetchmail release 6.3.18+NTLM+SSL+NLS.
>> [...]
>> CYGWIN_NT-5.2 server 1.7.7(0.230/5/3) 2010-08-31 09:58 i686 Cygwin
>>
>> Unfortunately, the server from where to fetch mails allows From:
>> addresses with non-ascii chars in the local part, i.e. dféßdfg at abc.de.
>> The server, which fetchmail sends its mails to, Exchange 2003, OTOH
>> doesn't accept these with 501 error.
>>
>> As all these mails are indeed spam, I tried to set "antispam 501" in
>> fetchmailrc. But apparently due to "set softbounce" being enabled by
>> default, even the deletion of mails matching the antispam setting aren't
>> deleted. I rather would like to leave softbounce enabled so that e.g.
>> oversized messages are still accessible through the web mailer, but
>> nonetheless should these antispam matching messages be deleted. Is this
>> possible somehow?
>
> No, softbounce overrides everything.
:-(

> Oversized messages aren't normally deleted, but left on the server
> anyways -- that is, unless you set --limitflush or --flush (both off by
> default).

I don't think of oversized messages due to some limit setting (I don't 
use a limit setting), but a smtp error from the receiving server that 
the mail is too big. In this case, it shouldn't (for me) be bounced 
and deleted, but instead left alone on the POP3 server. I think this 
is what softbounce is useful for.

So, apparently, (1) antispam and softbounce are (currently) not 
useable together, and (2) the limit setting is probably handled 
differently from the error condition that the smtp server rejects a 
message due to a size limit.  (2) might be reasonable because when 
using limit there is no transfer of the message to fetchmail.  But (1) 
seems misdesigned somehow.  Also: if softbounce overrides antispam, 
then it's confusing that the SMTP error (invalid From: address in my 
case) is only logged if antispam is not enabled. So antispam always 
influences log output, but is honored only if no softbounce.

Martin
