********** UPDATE **********

I think I have gotten it working. This is my current working model of
the ".fetchmailrc" file.

# Configuration created Sun Apr 24 12:17:09 2011 by fetchmailconf 1.57
set logfile "/var/log/fetchmail.log"
set idfile "/var/tmp/.fetchids"
set postmaster "postmaster"
set nobouncemail
set no spambounce
set no softbounce
set properties ""
poll imap.gmail.com with proto IMAP timeout 30 envelope Delivered-To
     localdomains seibercom.net
       user 'jerry at seibercom.net' there with password 'SecRet' is
       '*' here options forcecr dropdelivered ssl smtphost seibercom.net

Once I added the 'localdomains' data, it started delivering the
messages correctly; however, they were looping and being bounced by
Postfix. I added the 'dropdelivered' option and it worked. I have got
to give it some further testing; however, I feel that I might have
finally made a break through here.
