> The problem is that fetchmail isn't configured to pass the envelope
> address.

> 
> 1) set the Envelope option correctly
> 2) use "to * here"
>

OK, after making those changes all of the mail is now delivered to the
"Postmaster" address. this is the new .fetchmailrc file:

# Configuration created Sat Apr 23 19:04:10 2011 by fetchmailconf 1.57
set logfile "/var/log/fetchmail.log"
set idfile "/var/tmp/.fetchids"
set postmaster "postmaster"
set nobouncemail
set no spambounce
set no softbounce
set properties ""
poll imap.gmail.com with proto IMAP timeout 30 envelope Delivered-To and options no dns
       user 'jerry at domain.net' there with password 'SecRet' is '*' here options forcecr ssl
