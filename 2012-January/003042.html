well there are some reciepients who recieve mails from multiple boxes.
but then there are two sections in .fetchmailrc in that case. in example:

set postmaster "foo at bar.com"
set bouncemail
set properties "fetchall"
poll pop.foobar.com protocol POP3
        user foo at bar.com
        password foo
        mda "/usr/bin/zarafa-dagent foobar"
;


set postmaster "foo at bar.com"
set bouncemail
set properties "fetchall"
poll pop.foobar.com protocol POP3
        user foobar_private at hotmail.com
        password foobar
        mda "/usr/bin/zarafa-dagent foobar"
;

you see? the user foobar gets the mails from his private mailbox and from 
his company mailbox.
but he doesn't have to be able to send as the private user. because the user 
"foobar" has only one Mailadress in zarafa anyway.
Except he has some send as permissions from the info at foobar.com mailbox, but 
thats also handled by zarafa.

so i dont really know if i have multidrop per definition at all
