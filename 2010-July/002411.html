> I've just started using the sslfingerprint (as a result of this

> down, I've yet to see any errors related to invalid fingerprints.  I'm
> surprised to learn that a POP provider would issue different
> certificates with different fingerprints.  Even though there might be
> multiple IP's behind it, there's still only 1 "plus.pop.mail.yahoo.com",
> right?  I figured a fingerprint error would alert me, at the best, to a
> new certificate so that I might take the time to verify it myself.

They can share (copy) the certificate across all servers, or they can use  
an individual one. You don't usually care because clients are supposed to  
use the certificate's signature chain back to a trusted root signer, to  
establish validity -- and not fingerprints.

Usually the certificate authority (CA) certificates (the ones that you  
need to install into .../ssl/certs and that are trusted if you put them  
there) are issued for 5 or 10 years, and the CA can re-issue server  
certificates. These are often issued for 0.5 - 3 years only and frequently  
changed. Each time, the fingerprint will change, but the same root CA  
configuration can establish validity for the new certificate.
