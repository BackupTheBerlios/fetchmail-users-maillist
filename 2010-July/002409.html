Hi Gerry,

Fetchmail should complain rather noisily about incorrect certificates and  
refuse to connect to those servers that present invalid servers, but it  
does not loop retrying until it gets a good one - it has no idea that (or  
if so, how) load balancing is done on the other end unless they put  
multiple IPs into the DNS.  So --sslcertck might make things worse.   
--sslfingerprint is OK as long as you only get that one fingerprint back  
 from the certificates.  Is that the case?  I'd be somewhat surprised if it  
was.
