tracing what is served up by inbox, it is sufficient to have these two
certs installed
geo:MD5 Fingerprint=F7:75:AB:29:FB:51:4E:B7:77:5E:FF:05:3C:99:8E:F5
rapid:MD5 Fingerprint=1B:EE:28:5E:8F:F8:08:5F:79:CC:60:8B:92:99:A4:53
and then this print in fetchmailrc
inbox:MD5 Fingerprint=81:FB:6C:4F:52:9E:EB:20:A3:42:43:50:45:AA:62:E5

geo is in the current nss set, so if the system is up to date with that and
you're blindly accepting all those ca's (not the best idea), only rapid should
be needed locally.

no idea where equifax came from, maybe a mismatched c_rehash.
