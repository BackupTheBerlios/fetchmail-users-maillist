You should put each config item on a
separate line. And you should quote
things like usernames and passwords.
Read the man page again, the examples
that ship with it, and the internet search.
Then run it in debug mode to be sure.

I have abused the 'poll' string because fetchmail doesn't
seem to support my need for multiple accounts
on a provider and multiple providers in an rc.
And fetchmail wants to be a single process,
so I'll be hacking it all differently real soon now.
And getting rid of procmail :)

fetchmail -v -v -f fmrc -d 300 -N -K -a -n string
set invisible
poll username at example.com
 via imap.example.com
 service 993
 protocol imap
 authenticate password
 username "username at example.com"
 password "password"
 ssl
 sslcertck
 sslfingerprint "M:D:5"
 mda "procmail -m HOME=$HOME/mail/username at example.com $HOME/mail/procmailrc"
 keep
 folder "INBOX"


VERBOSE=on
LOGFILE=log
LOGABSTRACT=all
COMSAT=no
:0 c
Maildir_archive/
:0
