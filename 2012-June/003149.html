On Mon, Jun 4, 2012 at 6:53 AM, Mike Burns <mike at mike-burns.com> wrote:
> The big change since I wrote that blog post is that I no longer run my
> own mail server. I figured fetchmail could help me out here.

If your remote mail server supports POP3 and UIDL then you may find
that it works the way you're wanting. The trouble is that Fetchmail
isn't designed as a mail backup solution and expects to be the only
thing accessing a mailbox. It doesn't support marking emails as unread
after they've been fetched.
