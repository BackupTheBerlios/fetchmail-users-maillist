Am 06.11.2012 02:59, schrieb grarpamp:
> I'd like to run a fetch of lots of accounts at a host.
> I can set up the rc to get the host and other parameters.
> I can say on command line: --user <username>
> But I don't see anything like: --pass <password>
> In this case, it just doesn't make sense to
> edit into and bloat the rc with all the user/pass pairs.
> Am I missing an option?
> If not, this is a request for same :)

Greetings,

like Rob already wrote (thanks!), you are not missing an option, it was
deliberately omitted in fetchmail's design to avoid people sniffing on
other people's passwords by looking at /proc/ file systems or through
the ps/pgrep commands -- these usually expose the full command-line.

I am aware that there are systems that limit the visibility of process
information for unprivileged users, so the code for --password has not
been written, and possibly never will, because getting this right, and
limited to safe system configuration is an enormous and error-prone effort.

Sorry.

Best regards,
Matthias
