On Tue, Nov 6, 2012 at 1:59 AM, grarpamp <grarpamp at gmail.com> wrote:
> I'd like to run a fetch of lots of accounts at a host.
> I can set up the rc to get the host and other parameters.
> I can say on command line: --user <username>
> But I don't see anything like: --pass <password>
> In this case, it just doesn't make sense to
> edit into and bloat the rc with all the user/pass pairs.

Why not? Putting it on the command line means it has to be in a system
startup file if you want it to run on boot - a file that's likely to
be readable by all users. The fetchmail rc file is only readable by
the user running fetchmail (else fetchmail will refuse to run). Also,
command line arguments are generally visible to all users on the box
when they run the ps command.
