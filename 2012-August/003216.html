>> Quick background: I use Fastmail as my email server, which provides IMAP
>> access. I read my email using mutt, K-9, the Web interface, etc.
>>
>> I'd like to grab each email as soon as possible (in practice, faster
>> than I can read it) and commit it to a git repo.
>>
>> I've tried this using fetchmail but, due to a combination of the IMAP

> 
>> Any advice?
> 
> If you are willing to recompile fetchmail, apply the attached patch.

>

> as seen.
> 
> Perhaps someone could make this a configurable option ...

Robert,

thanks for the suggestion.  I must however say that I will not accept
the patch into the baseline code.

The proper solution is to track IMAP UID message numbers and a
mailbox/folder's UIDVALIDITY on fetchmail's end - just like it does for
POP3 with --uidl already. Only that UIDs are much shorter and grow
strictly monotonically, so it should be quicker than POP3 UIDLs (server
permitting, of course ;-)).

This has been on my TODO list for long, but I have not been able to make
much progress towards new major releases in the past 12 months.
