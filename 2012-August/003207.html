Sure.  Just meaning to say that you do should not end up with two double
quote characters next to each other :-)

(Technically speaking, it's not an argument, but an OR-list, where the
shell executes the second command, the :, if and only if the
left-hand-side of the || has exited with a nonzero status. And : happens
to be a shell command that eats all its own arguments end exits with
status 0.)

Well, documentation of this is often in the reference documentation of a
shell, which is not supposed to answer "what's this good for" kind of
questions.

Given that fetchmail uses the "system" library function to execute the
preconnect command line, it will be equivalent to your saying /bin/sh -c
..., hence all the shell tutorials compendia referring to Bourne- and
Korn compatible shells (like sh, ksh, pdksh, bash, ...; as opposed to
C-shells like csh/tcsh) apply (strictly speaking only the subset of it
that your shell supports).

So you'll get something along these lines, with --options replaced:

preconnect "exampleprogram --options arguments || :"

Which indeed has a pair of double quote characters. If it's too unobvious,

	... || true

or
	... || exit 0

are more explicit :)
