> Fact is, that to make it work, for me at least, is to gobble up any error

Well, suppressing any error message is as simple as adding "|| :" in the
script, or, more obviously "|| true", each time without the quotes.

The interface, as it stands, was originally designed so you could launch
something like

  preconnect "ssh -L12345:127.0.0.1:143 -luser host.example.org sleep 5"

so that you have an ssh tunnel where SSL wasn't available. In that case,
only "0" was a valid return code any anything else meant that the tunnel
was not available and should/could not be used.
