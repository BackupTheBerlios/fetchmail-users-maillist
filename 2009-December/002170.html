>     Here is my .fetchmailrc:

> poll internal protocol imap port 993
>  username "xxx" password "xxx" is "www-data" here ssl
>  mda "/usr/local/bin/email2trac --project=project --ticket_prefix=support"
> 
>     Note that "internal" is the DNS name of my IMAP mail server, and

>     When I run fetchmail, the SSL connection immediately fails but does

>     All it tells me is "SSL connection failed."  When I google that 
> message, I see that other people get other interesting messages about 
> things like certificates.
> 
>     I have verified that the IMAP server is working.  (I use the same 
> server for my personal inbox.)
> 
>     Why is the "--verbose" option not giving me any useful information?

Retry with 6.3.13. If the problem then persists, it's likely that the SSL
connection dies before SSL negotiation can make reasonable progress, often
caused by incomplete cert' info.

Please try the same connection with gnutls-cli or openssl s_client (see the
respective manual pages for details) and see if it makes progress.

HTH
