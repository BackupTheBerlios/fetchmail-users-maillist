> You may need to write one yourself, one that looks for Return-Path: and

You could try the one I wrote a couple of years ago for the exact
same purpose:

  http://devel.ringlet.net/textproc/trivsplitmail/

Of course, use it at your own risk :)  As it says in the README, just
run "trivsplitmail /path/to/mail.spool" (in your case, the path to
the "mailbox" file named "fetchmail"), and it will output a bunch of
files in your current working directory that you can either run
"procmail" on, or just move into a Maildir's new/ directory.

In a future version (whoah, that sounds grand for such a simple tool...)
I might add an output directory and Maildir-style file naming :)
