Geoff Powell schrieb:
> Hello,
> 
>  
> 
> I have fetchmail get mail from a pop3 server, call procmail to send it to a
> maildir folder but the mail gets created with 600 fetchmail:mail ower:group
> and I can't read it by default and have to do a chmod or chown manually.
> 
>  
> 
> Is there anyway to drop the mail into the maildir with either user as owner
> or 666 permissions?
> 
>  
> 
> fetchmail is running in daemon mode and is calling the following rc file:
> 
>  
> 
> poll mail.server.net proto pop3
> 
>   user username with password passwd is localuser here
> 
>   mda "/usr/bin/procmail -m /etc/procmailrc"
> 
>   fetchall
> 
>  
> 
>  
> 
> the procmailrc looks like this:
> 
>  
> 
>  MAILDIR=/var/mail/folders/user/
> 
>  
> 
>  :0
> 
>  .mailfolder/.
> 

Don't deliver from /etc/procmailrc - at the time, procmail hasn't dropped
privileges.

Also, be sure to read the procmail man page, paying particular attention to
the -m option. It is likely not what you want. You probably want -d instead.
