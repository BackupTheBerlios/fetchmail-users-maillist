On Fri, Nov 21, 2008 at 08:17, Boxhorn-EDV | Bernhard Boxhorn
<bernhard at boxhorn-edv.com> wrote:
> Hi fetchmail list,
>
> we´re using fetchmail for smaller servers to pull mails from the pop
> servers to their mailboxes, mostly when we don´t have control over the
> MX settings.
> To keep their mailboxes small and prevent the users for missusing e-mail
> for data transfers with large files we limit the local MTA to e.g. 10
> MB.
> Often the MX allows bigger messages. So if there are bigger messages we
> delete them with the --limitflush option, user gets a notification that
> a oversized message has been deleted.
> Everything fine until now.
>
> The Problem is: The user has no clue who was the original sender and the
> sender does not get any error message.
> So user knows there was a big message and it has been deleted and sender
> thinks user received the message, there is a lack of information.

The best approach for you is probably to configure the local MTA with
the size limit, but not configure fetchmail with limits.  That way
fetchmail will attempt the delivery and a bounce message will be
generated to the original sender.
