> Does this indicate a problem between fetchmail and the ISP, as I believe it does?

I don't think so. What happens is that fetchmail stuffs the mail down the
tube of whatever is your SMTP server (you removed that info, but have it on
the screen or in the logs) and that SMTP server took a while to digest the
message.

If there are online, inline, before-queue spam filtering mechanisms in
place, or you're directing at a spam/virus filter's front-end, that might
be explain it. Sometimes switching to after-queue filtering helps. It
doesn't add to the pain, since when using fetchmail, you're behind some
accept, store, forward scheme anyhow

It'll help fix 2 security and 2 critical bugs, but I don't think it will
solve your issue - so think about what problem you want to tackle most
urgently before delving deep into some action that doesn't help your
system's mail slowness.
