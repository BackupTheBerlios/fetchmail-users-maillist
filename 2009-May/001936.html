At first, thanks for your patience Rob :-)

I'm running debian lenny with kernel 2.6.26 and mail related software is:

- fetchmail 6.3.9~rc2
- procmail 3.22-16
- postfix 2.5.5-1.1

I'm running fetchmail as daemon (/etc/init.d/fetchmail). The config file 
/etc/defaults is:

# --- START /etc/defaults/fetchmail ---#
# This file will be used to declare some vars for fetchmail
#
# Uncomment the following if you don't want localized log messages
# export LC_ALL=C

# If you want to specify any additional OPTION to the start
# scripts specify them here
# OPTIONS=...

# Declare here if we want to start fetchmail. 'yes' or 'no'
START_DAEMON=yes
# --- END /etc/defaults/fetchmail ---#

The /etc/init.d/fetchmail daemon is quite long. I've pasted it there:

  http://pastebin.ca/1433237

For starting fetchmail I simply run:

  /etc/init.d/fetchmail start
